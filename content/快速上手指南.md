## 🚀 三步启动

### 第1步：创建GitHub仓库

1. 访问 https://github.com/new
2. 仓库名：`isp-course`（或你喜欢的名字）
3. 设为 **Public**（学生可访问）
4. **不要** 勾选 "Add a README"（我们已经有了）
5. 点击 "Create repository"

### 第2步：初始化Git

打开终端，运行：

```bash
cd "/Users/leiyu/Library/Mobile Documents/iCloud~md~obsidian/Documents/智能信号处理"

# 运行初始化脚本
./scripts/setup-git.sh https://github.com/你的用户名/isp-course.git
```

### 第3步：首次推送

```bash
# 提交所有文件
git commit -m "feat: 初始化智能信号处理课程仓库"

# 推送到GitHub
git push -u origin main

# 创建版本标签
./scripts/tag-version.sh v1.0.0 "2026秋季学期初始版本"
```

## 🏷️ 版本管理

### 版本号规则

| 类型 | 说明 | 何时使用 |
|------|------|----------|
| `v1.0.0` → `v1.1.0` | 新增知识点/项目 | 添加新内容 |
| `v1.1.0` → `v1.1.1` | 修复错误 | 修正描述、错字 |
| `v1.x.x` → `v2.0.0` | 重大调整 | 下学期课程改革 |

### 创建版本

```bash
# 交互式创建
./scripts/tag-version.sh v1.1.0 "新增深度学习图像处理章节"

# 或直接指定
./scripts/tag-version.sh v1.2.0 "添加跨学科项目E3鸟类监测"
```

## ❓ 常见问题

### Q: iCloud文件同步慢怎么办？

```bash
# 触发文件下载
brctl download "/path/to/file"

# 或等待iCloud自动同步
```

### Q: 推送到GitHub失败？

```bash
# 拉取最新版本
git pull origin main

# 解决冲突后重新推送
git push origin main
```

### Q: 如何回滚到旧版本？

```bash
# 查看所有标签
git tag -l

# 查看某个版本的内容
git show v1.0.0

# 回到某个版本（创建新分支）
git checkout -b restore v1.0.0
```

### Q: 学生能看到我的私人笔记吗？

不会。`.gitignore` 已配置排除：
- `.obsidian/` — 你的Obsidian配置
- `private/` — 私人文件夹
- `drafts/` — 草稿文件夹
