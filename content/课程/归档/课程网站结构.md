## 🧭 学生使用指南

### 场景1：我想了解课程整体框架
→ 打开 `欢迎.md` → 看 `课程体系架构图.png`

### 场景2：我想学习某个知识点
→ 打开 `知识点/模块名/知识点名.md`
→ 每个文件包含：核心内容 + 公式 + 配图 + 教材章节 + 视频链接

### 场景3：我要做项目，遇到不懂的理论
→ 查看 `课程/项目说明书_xxx.md` 中的"参考知识点"列
→ 点击对应知识点的wiki-link跳转学习

### 场景4：我想深入学习某个方向
→ 知识点文件中的 `## 扩展阅读` 部分
→ 📖 教材对应章节 → 🎥 视频资源 → 🔗 相关知识点

### 场景5：我想知道考试考什么
→ `课程/考核评分标准.md`

## 📊 知识点文件标准格式

每个知识点md文件应包含：

```markdown
# 知识点名称

## 一、定义与概念
（直觉性描述，配合可视化）

## 二、核心公式
（关键公式，用LaTeX标注）

## 三、典型应用
（Mermaid流程图 + 应用案例）

## 四、参考文献
（3-5篇核心论文/教材）

## 扩展阅读
### 📖 教材对应章节
### 🎥 视频资源
### 🔗 相关知识点

## 相关知识点
（Obsidian wiki-links）
```
