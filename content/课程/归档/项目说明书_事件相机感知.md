# 项目说明书：事件相机信号处理与场景感知

## 一、项目背景

事件相机（Event Camera / Dynamic Vision Sensor, DVS）是一种受生物视网膜启发的新型视觉传感器。与传统帧式相机不同，事件相机的每个像素独立工作：当像素亮度变化超过阈值时，异步输出事件 $e = (x, y, t, p)$，其中 $(x,y)$ 为像素坐标，$t$ 为微秒级时间戳，$p$ 为极性（亮度增/减）。

这一设计带来了独特优势：
- **超高时间分辨率**：微秒级时间精度（传统相机约30fps）
- **超高动态范围**：>120 dB（传统相机约60 dB）
- **极低延迟与功耗**：仅在变化时输出数据
- **无运动模糊**：不依赖曝光时间

事件相机的研究融合了**计算机视觉**与**神经科学**：其设计灵感源自生物视网膜的ON/OFF通路，而其数据处理则需要全新的算法框架。DVS（Dynamic Vision Sensor）研究组在苏黎世联邦理工学院(ETH Zurich)等机构推动了该领域的快速发展。

本项目引导学生掌握事件相机数据的特性，学习事件流的表征与处理方法，并应用深度学习实现图像重建或光流估计等任务。

### 跨学科知识点

| 领域 | 关键概念 |
|------|---------|
| 神经科学 | 视网膜ON/OFF通路、时间对比检测、稀疏编码 |
| 计算机视觉 | 光流估计、图像重建、视觉里程计 |
| 信号处理 | 异步采样、时间序列分析、事件流滤波 |
| 深度学习 | 3D卷积、事件体素表征、自监督学习 |

1. 理解事件相机的工作原理与数据特性
2. 掌握事件数据的加载、可视化与预处理方法

## 三、数据集

### 主数据集 A：MVSEC

**Multi Vehicle Stereo Event Camera Dataset**

- **网址**：http://dsec.ifi.uzh.ch/mvsec/
- **下载地址**：https://github.com/daniilidis-group/mvsec
- **场景**：室内飞行、室外驾驶等
- **内容**：双目DVS事件流 + 灰度帧 + IMU + GT光流/深度
- **格式**：HDF5
- **特点**：提供光流和深度真值，适合光流估计任务

```python
# MVSEC 数据结构示例
# /davis/left/events  — 事件流 (x, y, t, polarity)
# /davis/left/image_raw  — 灰度帧
# /davis/left/flow_dist  — GT 光流
```

### 主数据集 B：DSEC

**Dynamic Stereo Event Camera Dataset**

- **网址**：http://dsec.ifi.uzh.ch/
- **下载地址**：http://dsec.ifi.uzh.ch/dsec-download/
- **场景**：城市驾驶（苏黎世市区）
- **内容**：双目DVS事件流 + 灰度帧 + GT光流
- **格式**：HDF5
- **特点**：大规模城市驾驶场景，光流真值质量高

## 五、项目阶段与任务

### 阶段一：事件数据加载与可视化（第1-2周）

**目标**：掌握事件数据的基本结构，学会加载与可视化

**任务清单**：

1. **数据加载**
   - 使用 `h5py` 读取 MVSEC 或 DSEC 的 HDF5 文件
   - 解析事件数据字段：`x, y, timestamp, polarity`
   - 记录关键元数据：传感器分辨率、时间范围、事件总数

2. **事件流统计分析**
   - 绘制事件速率（每秒事件数）随时间的变化曲线
   - 分析正/负极性事件的比例与空间分布
   - 计算事件时间间隔分布

3. **基础可视化**
   - **事件帧(Event Frame)**：将指定时间窗口内的事件累积为图像
     ```python
     def events_to_frame(events, sensor_size, dt):
         frame = np.zeros(sensor_size)
         for e in events:
             frame[e['y'], e['x']] += (2 * e['p'] - 1)  # +1/-1
         return frame
     ```
   - **时间表面(Time Surface)**：每个像素记录最近一次事件的时间
   - **事件动画**：生成滑动窗口的事件可视化动画

4. **使用 tonic 进行可视化**
   - 使用 `tonic.transforms.ToFrame` 生成事件帧序列
   - 尝试不同的时间窗口/bins数量，观察效果差异
   - 使用 `tonic.utils.plot_event_grid` 进行可视化

**交付物**：
- 数据加载与可视化 Notebook
- 统计分析图表
- 事件可视化动画（.gif 或 .mp4）

---

### 阶段二：事件流处理与表征学习（第3-5周）

**目标**：学习事件数据的多种表征方式，理解其优缺点

**任务清单**：

1. **表征方法实现**（至少实现3种）

   - **体素网格 (Voxel Grid)**
     将事件流离散化为时空3D体素，每个体素累计极性加权事件
     ```python
     def events_to_voxel(events, sensor_size, n_bins):
         voxel = np.zeros((n_bins, *sensor_size))
         # 按时间分bin，线性插值分配到相邻bin
         ...
         return voxel
     ```
   
   - **事件帧 (Event Frame)**
     正负极性事件分别累积或合并为单通道/双通道图像

   - **时间表面 (Time Surface / SAP)**
     表面活跃度模式(Surface of Active Pixels)

   - **事件直方图 (Event Histogram)**
     类似事件帧但使用对数或归一化

   - **Ternary/N-ary Representation**
     使用多个时间bin的三值化表征

2. **表征对比分析**
   - 对同一段事件流使用不同表征方法
   - 可视化对比各表征的信息保留情况
   - 讨论信息损失与计算效率的trade-off

3. **基于事件表征的简单任务**
   - 使用事件帧 + 传统方法（如光流法）估计运动
   - 或使用事件表征 + 简单CNN进行分类（如 MNIST-DVS 数据集）

4. **事件去噪**
   - 实现基于时空邻域的事件滤波方法
   - 去除背景噪声事件（hot pixels、spurious events）
   - 对比去噪前后的表征质量

**交付物**：
- 事件表征 Notebook（含多种表征实现）
- 表征对比可视化
- 去噪效果对比图

---

### 阶段三：图像重建或光流估计（第6-10周）

**目标**：应用深度学习完成一个完整的感知任务

学生可从以下两个方向**选择一个**完成：

#### 方向 A：从事件流重建图像

**任务清单**：

1. **网络设计**
   - 基于 U-Net 或类似编解码架构
   - 输入：事件表征（体素网格或事件帧堆叠）
   - 输出：灰度/彩色图像
   - 参考论文：E2VID (Rebecq et al., CVPR 2019)

2. **训练流程**
   - 数据预处理：滑动窗口提取事件段，配对灰度帧作为GT
   - 数据增强：随机裁剪、翻转、时间抖动
   - 损失函数：L1/L2 + 感知损失(perceptual loss) + 时序一致性损失
   - 优化器：Adam, 学习率调度

3. **评估**
   - 定量指标：PSNR、SSIM
   - 定性评估：视觉质量对比
   - 对比不同表征方法对重建质量的影响

#### 方向 B：从事件流估计光流

**任务清单**：

1. **网络设计**
   - 基于编解码架构（如 FlowNet 的事件版本）
   - 输入：事件体素网格
   - 输出：密集光流场 (u, v)
   - 参考论文：EV-FlowNet (Zhu et al., IROS 2018)

2. **训练流程**
   - 数据加载：使用 MVSEC/DSEC 的 GT 光流
   - 损失函数：端点误差(EPE) + 平滑正则化
   - 自监督选项：基于事件的亮度恒定约束

3. **评估**
   - 定量指标：平均端点误差(AEE)、Fl-all
   - 可视化：光流场着色(flow coloring)
   - 与帧式光流方法(如 RAFT)对比

#### 方向 C（加分项）：事件相机视觉里程计

- 利用事件流实现简单的位姿估计
- 需要结合 IMU 数据（MVSEC 提供）
- 可使用事件特征点匹配或直接回归方法

**交付物**：
- 完整的训练/测试代码
- 训练日志与曲线
- 可视化结果（重建图像或光流场）
- 与 baseline 方法的对比分析
