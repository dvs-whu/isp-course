## 一、版本控制策略

### 版本号规则

```
v{主版本}.{次版本}.{修订号}
```

| 类型 | 说明 | 示例 |
|------|------|------|
| 主版本 (Major) | 课程大纲重大调整、学期更换 | v2.0.0 |
| 次版本 (Minor) | 新增知识点、项目、实验 | v1.3.0 |
| 修订号 (Patch) | 错误修复、措辞优化、格式调整 | v1.2.1 |

### 标签策略

```bash
# 学期标签
git tag -a v1.0-fall2026 -m "2026秋季学期课程版本"

# 周次标签（每周结束时）
git tag -a week1 -m "W1完成：信号概述、环境搭建、项目一选题"
git tag -a week2 -m "W2完成：傅里叶分析、频谱解读"

# 里程碑标签
git tag -a project1-due -m "项目一截止"
git tag -a midterm -m "期中节点"
```

## 四、自动化发布脚本

### 发布脚本 `scripts/publish.sh`

```bash
#!/bin/bash
# 智能信号处理课程发布脚本

set -e

VAULT_DIR="/Users/leiyu/Library/Mobile Documents/iCloud~md~obsidian/Documents/智能信号处理"
cd "$VAULT_DIR"

# 颜色定义
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

echo -e "${GREEN}📚 智能信号处理课程发布工具${NC}"
echo "================================"

# 检查是否有未提交的更改
if [[ -n $(git status -s) ]]; then
    echo -e "${YELLOW}⚠️  检测到未提交的更改:${NC}"
    git status -s
    echo ""
    read -p "是否提交这些更改？(y/n) " -n 1 -r
    echo
    if [[ $REPLY =~ ^[Yy]$ ]]; then
        read -p "请输入提交信息: " commit_msg
        git add -A
        git commit -m "$commit_msg"
    fi
fi

# 显示当前版本
echo ""
echo "当前版本标签:"
git tag -l "v*" --sort=-v:refname | head -5

echo ""
read -p "是否创建新版本标签？(输入版本号如 v1.1.0，或按回车跳过) " version
if [[ -n "$version" ]]; then
    read -p "版本说明: " version_msg
    git tag -a "$version" -m "$version_msg"
    echo -e "${GREEN}✅ 已创建标签: $version${NC}"
fi

# 推送到GitHub
echo ""
echo "推送到GitHub..."
git push origin main
git push origin --tags

echo -e "${GREEN}✅ 发布完成！${NC}"
echo ""
echo "GitHub仓库: https://github.com/dvs-whu/isp-course"
```

## 八、最佳实践

### ✅ 推荐做法

1. **频繁提交**：小改动也要提交，保持历史清晰
2. **有意义的提交信息**：`feat:` 新功能、`fix:` 修复、`docs:` 文档
3. **使用标签**：重要节点打标签，方便回溯
4. **分支开发**：大改动用分支，避免污染main
5. **定期推送**：每天或每周推送到GitHub

### ❌ 避免做法

1. **大文件**：视频、大型数据集用Git LFS或外部链接
2. **敏感信息**：不要提交学生成绩、密码等
3. **频繁合并冲突**：多人协作时注意文件分区
4. **忽略.gitignore**：确保临时文件不被提交

## 九、故障排除

### iCloud同步问题

如果遇到 "Resource deadlock avoided" 错误：

```bash
# 触发文件下载
brctl download "/path/to/file"

# 或者等待iCloud同步完成
```

### Git推送失败

```bash
# 拉取最新版本
git pull origin main

# 解决冲突后重新推送
git push origin main
```

### 标签冲突

```bash
# 删除本地标签
git tag -d tagname

# 删除远程标签
git push origin --delete tagname

# 重新创建
git tag -a tagname -m "message"
git push origin tagname
```
