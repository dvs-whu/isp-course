## 2. 通俗入口

> [!tip] 🎯 一句话理解
> 自适应滤波器就像一个"会自己调参数的滤波器"——你不需要提前知道信号的全部信息，它会边工作边学习，自动找到最佳参数。

**一个真实问题**：你戴降噪耳机坐飞机。引擎噪声的频率和强度随飞行状态变化，不可能提前设计一个固定滤波器。自适应滤波器会实时"听"噪声，自动调整参数来生成反相声波。

**生活类比——调收音机**：信号时强时弱、电台频率会漂移，你需要不断微调旋钮才能听到清晰的声音。LMS 算法就是那个"慢慢拧旋钮"的策略——每次偏差大就多拧一点，偏差小就少拧一点。

**学这个知识点能解决什么**：
- 主动降噪耳机：参考麦克风采噪声 + LMS = 实时反相抵消
- 回声消除：电话/会议系统中消除声学回声
- 信道均衡：通信接收端自动补偿信道失配

## 3. 严谨核心

### 数学对象

自适应滤波器是一种能够根据输入信号的统计特性自动调整滤波器参数的系统。当信号环境未知或时变时，自适应滤波器无需事先知道信号和噪声的统计特性，而是通过递推算法在线学习最优滤波器参数。

### 基本假设

- **不需要信号和噪声的先验统计知识**（与维纳滤波的核心区别）
- 从初始条件出发，利用观测数据逐步逼近维纳滤波器的最优解
- 算法的每一步都根据当前误差调整权向量，使目标函数逐步减小

### 核心公式

**FIR自适应滤波器输出：**

$$y[n] = \mathbf{w}^H[n]\mathbf{x}[n] = \sum_{k=0}^{M-1} w_k^*[n] x[n-k]$$

**误差信号：**

$$e[n] = d[n] - y[n] = d[n] - \mathbf{w}^H[n]\mathbf{x}[n]$$

**LMS算法权更新：**

$$\mathbf{w}[n+1] = \mathbf{w}[n] + \mu \cdot \mathbf{x}[n] e^*[n]$$

其中 $\mu$ 为步长参数，收敛条件为 $0 < \mu < \frac{2}{\lambda_{max}}$，$\lambda_{max}$ 为 $\mathbf{R}$ 的最大特征值。

**NLMS算法权更新：**

$$\mathbf{w}[n+1] = \mathbf{w}[n] + \frac{\mu}{\|\mathbf{x}[n]\|^2 + \delta} \mathbf{x}[n] e^*[n]$$

**RLS算法：**

$$\mathbf{k}[n] = \frac{\lambda^{-1}\mathbf{P}[n-1]\mathbf{x}[n]}{1 + \lambda^{-1}\mathbf{x}^H[n]\mathbf{P}[n-1]\mathbf{x}[n]}$$

$$\mathbf{w}[n] = \mathbf{w}[n-1] + \mathbf{k}[n]e^*[n]$$

$$\mathbf{P}[n] = \lambda^{-1}\mathbf{P}[n-1] - \lambda^{-1}\mathbf{k}[n]\mathbf{x}^H[n]\mathbf{P}[n-1]$$

其中 $\lambda$ 为遗忘因子（$0 < \lambda \leq 1$），$\mathbf{P}$ 为逆相关矩阵的递推估计。

### LMS 与 RLS 算法对比

| 特性 | LMS | NLMS | RLS |
| ------|-----|------|-----|
| 更新准则 | 随机梯度下降 | 归一化随机梯度 | 最小二乘 (指数加权) |
| 每步计算量 | O(M) | O(M) | O(M²) |
| 收敛速度 | 慢 | 中等 | 快 |
| 收敛条件 | 0 < μ < 2/λ_max | 0 < μ < 2 | 0 < λ ≤ 1 |
| 稳态失调 | 较大 | 中等 | 小 |
| 跟踪能力 | 弱 | 中等 | 强 |
| 数值稳定性 | 好 | 好 | 需注意 (矩阵正定) |
| 适用场景 | 统计平稳、计算资源有限 | 通用 | 快速收敛、时变环境 |
| 核心参数 | 步长 μ | 步长 μ, 正则化 δ | 遗忘因子 λ, 初始 P |

### 适用条件和边界

- LMS 收敛慢但计算简单，适合统计平稳、计算资源有限的场景
- RLS 收敛快但计算量大（$O(M^2)$），适合快速收敛和时变环境
- 步长 $\mu$ 过大会导致不稳定甚至发散；过小会收敛太慢
- LMS 存在"失调量"——稳态时权向量在最优解附近波动，这是随机梯度下降的固有特性

## 4. 方法流程

**输入**：
- 主信号 $d[n]$（含期望信号 + 噪声）
- 参考信号 $x[n]$（与噪声相关的参考通道）
- 滤波器长度 $M$、步长 $\mu$

**处理步骤**：
1. **初始化**：权向量 $\mathbf{w}[0] = \mathbf{0}$（或随机小值）
2. **滤波**：计算输出 $y[n] = \mathbf{w}^H[n]\mathbf{x}[n]$
3. **误差**：$e[n] = d[n] - y[n]$
4. **更新**：$\mathbf{w}[n+1] = \mathbf{w}[n] + \mu \cdot \mathbf{x}[n] e^*[n]$
5. 重复步骤 2-4，逐样本递推

**输出解释**：
- $e[n]$：误差信号，收敛后趋近于纯净信号（在噪声消除场景中）
- $\mathbf{w}[n]$：权向量，收敛后趋近于维纳解

**常见参数选择**：
- 步长 $\mu$：通常取 $0.01 \sim 0.1$（归一化后）；太大不稳定，太小收敛慢
- 滤波器长度 $M$：取决于噪声路径的复杂度；太小无法建模，太大增加计算量和失调
- 遗忘因子 $\lambda$（RLS）：接近 1 时记忆长，适合平稳环境；较小时遗忘快，适合时变环境

```python
# === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

# LMS 自适应滤波器：噪声消除示例
np.random.seed(42)
N = 1000
n = np.arange(N)

# 纯净信号和噪声
s = np.sin(2 * np.pi * 0.02 * n)       # 低频纯净信号
noise = 0.8 * np.random.randn(N)       # 噪声
d = s + noise                           # 主信号（含噪）
x_ref = noise + 0.2 * np.random.randn(N)  # 参考噪声通道

# LMS 算法
M = 16       # 滤波器长度
mu = 0.01    # 步长参数（太大→不稳定，太小→收敛慢）
w = np.zeros(M)  # 权向量初始化
e = np.zeros(N)  # 误差信号

for i in range(M, N):
    x_vec = x_ref[i-M+1:i+1][::-1]  # 参考输入
    y = np.dot(w, x_vec)             # 滤波器输出
    e[i] = d[i] - y                  # 误差 = 期望 - 输出
    w = w + mu * x_vec * e[i]        # LMS 权值更新

fig, axes = plt.subplots(3, 1, figsize=(10, 8))
axes[0].plot(n, s, 'b-')
axes[0].set_title('纯净信号（恢复目标）')

axes[1].plot(n, d, 'r-', alpha=0.6)
axes[1].set_title('Noisy Signal d[n] = s[n] + Noise')

axes[2].plot(n, s, 'b-', label='纯净信号', alpha=0.5)
axes[2].plot(n, e, 'g-', label='LMS 输出（误差信号）', alpha=0.8)
axes[2].set_title('LMS 自适应滤波：误差信号趋近纯净信号')
axes[2].set_xlabel('样本序号 n')
axes[2].legend()

plt.tight_layout()
plt.show()
```

**结果解释**：
- 误差信号 $e[n]$ 最终趋近于纯净信号——这就是自适应噪声消除的原理
- 初始阶段（前 100-200 个样本）误差较大，这是 LMS 收敛的过程
- 权向量 $w$ 从零开始，逐样本更新，最终收敛到维纳解附近

## 6. 常见误区

| 易错点 | 正确理解 |
|--------|----------|
| 步长越大收敛越快就越好 | 步长过大会导致不稳定甚至发散（需满足 $0<\mu<2/\lambda_{\max}$）；要在收敛速度和稳态误差之间权衡 |
| 自适应滤波器总能精确收敛到维纳解 | LMS 存在"失调量"——稳态时权向量在最优解附近波动，这是随机梯度下降的固有特性 |
| 自适应滤波和维纳滤波是完全不同的方法 | 自适应滤波是维纳滤波的在线实现——目标相同，只是求解方式不同（批处理 vs 递推） |
| LMS 算法中误差 e[n] 越小越好 | e[n] 小不等于 w 已收敛；e[n] 还受噪声影响，需要看 w 的收敛曲线而非只看 e[n] |

## 7. 启发问题与创新拓展

**一个反例或边界问题**：
如果参考噪声通道 $x[n]$ 和主信号中的噪声不相关，LMS 会怎样？——权向量无法收敛到正确的噪声模型，自适应滤波退化为"白减"，甚至可能引入新的噪声。这说明自适应滤波的前提是参考通道必须与噪声相关。

**一个跨学科迁移场景**：
LMS 的权更新规则 $\mathbf{w}[n+1] = \mathbf{w}[n] + \mu \cdot \mathbf{x}[n] e^*[n]$ 本质是随机梯度下降；Adam、AdaGrad 等深度学习优化器可以看作"自适应步长 + 动量"的推广——理解这条线索有助于建立信号处理与机器学习的统一视角。

**一个可做成课程项目的小想法**：
实现一个"变步长 LMS"：前期用大步长快速收敛，后期用小步长精细追踪。对比固定步长 LMS，观察收敛速度和稳态误差的改善。
