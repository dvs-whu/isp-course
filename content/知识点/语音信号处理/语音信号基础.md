> 3. **语谱图 = 语音的"指纹"**：横轴Time、纵轴Frequency、颜色深浅=能量，一眼看出语音结构
> 
> **作为进阶了解的（期末不要求推导）：**
> - LPC（线性PredictionCoding）的数学推导
> - 声道传输函数 $H(z)$ 的全极点建模
> 
> 💡 **一句话记忆法**：语音 = "声带怎么振 × 嘴巴怎么变"，基频是音高，共振峰是口型的名片！

## 3. 严谨核心

### 语音产生模型

> 语音信号的产生可用"激励源-声道Filtering器"线性Model $s(n) = e(n) * h(n)$ 描述。

### 定义与概念

语音Signal是人类通过声带振动和声道共鸣产生的时变信号，本质为一维声压随时间变化的波形。语音信号处理旨在利用数字信号处理Method对语音进行分析、Coding、合成和识别。

核心概念：
- **声学模型**：语音信号的产生可用"激励源-声道Filter"Model描述
- **短时平稳性**：语音Signal在10-30ms的短时窗内可近似为平稳随机过程
- **基音Frequency（F0）**：声带振动的基频，决定音高
- **共振峰（Formant）**：声道共振频率，决定音色
- **语音三要素**：音高、音强、音色

### 核心公式

语音信号的线性产生模型：

$$s(n) = e(n) * h(n)$$

其中 $e(n)$ 为激励信号，$h(n)$ 为声道冲激响应。

短时能量：

$$E_m = \sum_{n=-\infty}^{\infty} [s(n) w(m-n)]^2$$

其中 $w(n)$ 为窗Function（常用汉明窗）：

$$w(n) = 0.54 - 0.46 \cos\left(\frac{2\pi n}{N-1}\right), \quad 0 \le n \le N-1$$

短时自相关函数：

$$R_m(k) = \sum_{n=-\infty}^{\infty} s(n) w(m-n) \cdot s(n+k) w(m-n-k)$$

线性PredictionCoding（LPC）模型：

$$\hat{s}(n) = \sum_{i=1}^{p} a_i s(n-i)$$

LPC系数通过Minimize预测误差 $e(n) = s(n) - \hat{s}(n)$ 的均方值得到。

### 语音信号Feature参数对比

| 特征参数 | 物理意义 | 频率范围 | 提取方法 | 典型应用 |
| ---------|---------|---------|---------|---------|
| 基音频率 F0 | 声带振动基频 | 男 80-200Hz，女 150-350Hz | 自相关法、倒谱法 | 语音合成、说话人识别 |
| 共振峰 F1-F4 | 声道共振频率 | 200Hz-4kHz | LPC 谱峰值检测 | 语音识别、语音合成 |
| MFCC | 模拟人耳听觉特性 | 全频带 | Filtering器组 + DCT | ASR 主流特征 |
| 短时能量 | 信号Amplitude的时变特征 | — | 窗内平方求和 | 端点检测、VAD |
| 短时过零率 | 信号过零次数 | — | 符号变化计数 | 清浊音判断 |

## 4. 方法流程

**输入是什么：** 一段语音波形信号，采样率通常为 8kHz（电话）或 16kHz（宽带语音）。

**处理步骤：**
1. **预加重**：高通滤波补偿高频衰减，提升高频细节
2. **分帧加窗**：将语音切分为 10-30ms 的短时帧，每帧加汉明窗
3. **特征提取**：计算短时能量、过零率、F0、共振峰、MFCC 等特征
4. **端点检测（VAD）**：根据能量和过零率判断语音段与静音段
5. **后续任务**：将特征送入声学模型（ASR）、分类器（说话人识别）或生成模型（TTS）

**输出如何解释：** F0 序列反映音高变化轨迹；共振峰轨迹反映元音发音过程；MFCC 向量作为语音帧的压缩声学表示。

**典型应用：**
- 语音识别（ASR）系统中的特征提取（MFCC、滤波器组）
- 语音编码（如G.711、G.729等通信编码标准）
- 语音合成（TTS）中的声学参数生成
- 说话人识别与Validation

```python
# === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

import numpy as np
import matplotlib.pyplot as plt
from scipy.signal import lfilter, hamming

# 模拟语音Signal：激励源 × 声道Filter模型
fs = 8000  # Sampling率 8kHz
duration = 0.1  # 100ms
t = np.arange(0, duration, 1/fs)

# 激励信号：浊音用周期脉冲串，清音用白噪声
f0 = 120  # 基频 120Hz（男声）
excitation = np.zeros_like(t)
pulse_period = int(fs / f0)
for i in range(0, len(t), pulse_period):
    excitation[i] = 1.0

# 声道Filtering器：全极点模型（2个共振峰）
# F1=500Hz, F2=1500Hz 模拟元音 /a/
from scipy.signal import iirpeak
b1, a1 = iirpeak(500/(fs/2), Q=10)
b2, a2 = iirpeak(1500/(fs/2), Q=10)
vocal_tract = lfilter(b2, a2, lfilter(b1, a1, excitation))
speech = vocal_tract / np.max(np.abs(vocal_tract))  # 归一化

# 短时分析Parameter
frame_len = int(0.025 * fs)  # 25ms帧长
frame_shift = int(0.010 * fs)  # 10ms帧移
n_frames = (len(speech) - frame_len) // frame_shift + 1

# 计算短时能量
energy = np.zeros(n_frames)
for i in range(n_frames):
    start = i * frame_shift
    frame = speech[start:start+frame_len]
    energy[i] = np.sum(frame**2)

# LPC谱包络估计（p阶）
p = 12  # LPC阶数
frame_idx = n_frames // 2  # 取中间帧
start = frame_idx * frame_shift
frame = speech[start:start+frame_len] * hamming(frame_len)
# 自相关法求LPC
from scipy.linalg import toeplitz
r = np.correlate(frame, frame, mode='full')
r = r[frame_len-1:frame_len-1+p+1]
a_lpc = np.linalg.solve(toeplitz(r[:-1]), r[1:])
# LPCSpectrum
w_lpc = np.linspace(0, np.pi, 512)
H = 1.0 / np.abs(1 - np.sum(a_lpc * np.exp(-1j * w_lpc * np.arange(1, p+1)), axis=0))

# 绘图
fig, axes = plt.subplots(3, 1, figsize=(10, 8))
axes[0].plot(t*1000, speech, 'b-', linewidth=0.5)
axes[0].set_title('语音波形（元音 /a/ 的合成Signal）')
axes[0].set_xlabel('Time (ms)')
axes[0].set_ylabel('Amplitude')

t_energy = np.arange(n_frames) * frame_shift / fs * 1000
axes[1].plot(t_energy, 10*np.log10(energy+1e-10), 'r-')
axes[1].set_title('短时能量')
axes[1].set_xlabel('Time (ms)')
axes[1].set_ylabel('Energy (dB)')

spectrum = np.abs(np.fft.fft(frame, 1024))[:512]
freqs = np.linspace(0, fs/2, 512)
axes[2].plot(freqs, 20*np.log10(spectrum+1e-10), 'gray', alpha=0.5, label='Spectrum')
axes[2].plot(freqs, 20*np.log10(H*np.max(spectrum)/np.max(H)), 'r-', linewidth=2, label='LPC包络')
axes[2].set_title('LPC谱包络Estimation（红色）')
axes[2].set_xlabel('Frequency (Hz)')
axes[2].set_ylabel('Amplitude (dB)')
axes[2].legend()
axes[2].set_xlim([0, 4000])
plt.tight_layout()
plt.show()
```

> 运行环境：`numpy`, `scipy`, `matplotlib`。代码演示了语音Signal的**波形可视化**、**语谱图绘制**和**LPC 频谱包络Estimation**。

## 6. 常见误区

| 易错点 | 正确理解 |
|--------|----------|
| 语音只需要看波形 | 波形能看幅度和周期，但音素结构、共振峰和NoiseDistribution更容易在频谱/语谱图中观察 |
| F0和共振峰是一回事 | F0主要来自声带振动，决定音高；共振峰来自声道共鸣，反映口型和元音线索 |
| 语音全程平稳 | 语音整体非平稳，只能在10-30ms短窗内近似平稳 |
| MFCC就是语音内容 | MFCC是声学特征，不等同于文字内容；识别还需要声学模型、语言模型或端到端模型 |

**公式误解：** LPC的"线性预测"不是简单外推，而是利用语音信号的短时相关性建模声道滤波器，阶数p通常取10-12对应声道共振峰数量。

**参数误解：** 窗长不是越长越好——太长违反短时平稳假设，太短频率分辨率不足。25ms窗长和10ms帧移是经验最佳平衡。

**适用条件误解：** 激励源-声道模型假设声带振动与声道滤波线性解耦，对爆破音和摩擦音（清音）的解释力较弱。

## 7. 启发问题与创新拓展

**一个反例或边界问题：**
为什么语音识别通常不用Primal波形直接做人工Feature？因为波形中基频和共振峰在时域高度混叠，只有通过时频分析（语谱图）或听觉模拟（MFCC）才能把不同层次的信息解耦开来。

**一个跨学科迁移场景：**
动物声音监测（如鸟鸣、鲸歌）的声学结构与人类语音完全不同——没有声道模型、没有共振峰概念，但短时分析框架和时频特征提取的思路仍然适用。这说明"分帧加窗+时频表示"是语音信号处理的通用范式，不局限于人类语音。

**一个可做成课程项目的小想法：**
在项目一D中，采集不同说话人发同一元音（如/a/、/i/、/u/）的语音，提取MFCC特征后用PCA可视化——观察不同元音的MFCC是否自然聚类，以及同一元音不同说话人的分散程度。这能直观验证MFCC对音素的区分能力。

**其他启发问题：**
- 同一个人用不同口型发元音时，F0和共振峰谁变化更明显？
- 窗长太短或太长分别会Loss什么信息？
- 说话人、噪声和录音设备会怎样影响MFCC？
