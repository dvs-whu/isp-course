    卡尔曼滤波¶

    1. 学习定位¶

    [!note] 📍 学习导航 - 所属模块：统计信号处理 → 递推最优估计 - 周次：W7 - 学习深度：会用即可，理解预测-更新循环和卡尔曼增益的物理意义即可 - 先修知识：线性代数（矩阵运算）、概率论（高斯分布、条件期望）、随机信号基础（状态空间模型） - 关联实验：实验4（统计信号处理） - 关联项目：项目一E（目标跟踪与状态估计） - 核心问题：如何在有噪声的观测下，实时递推地估计动态系统的状态？ - 严谨边界：重点掌握预测-更新两步循环、卡尔曼增益的物理意义、协方差矩阵的作用；EKF/UKF和协方差递推推导作为了解 - 启发思考：卡尔曼增益 K 随时间变化说明了什么？初始协方差设得太大或太小会怎样？

    2. 通俗入口¶

    [!tip] 🎯 一句话理解 卡尔曼滤波就是"带记忆的加权平均"——每一步都根据模型预测和实际测量，按可信度加权融合，不断更新最优估计。

    一个真实问题：你开车用 GPS 导航。GPS 每秒告诉你一个位置，但有误差（几十米）。同时你知道车速和方向（运动模型）。卡尔曼滤波就是把这两种信息"融合"起来：GPS 可信时多信 GPS，GPS 信号差时多信运动模型。

    生活类比： 1. 预测：根据上一次的位置和速度，猜这次的位置（就像根据昨天的位置猜今天的位置） 2. 观测：用传感器测量这次的位置（GPS 告诉你现在在哪） 3. 更新：把猜测和测量加权平均——哪个更可靠就给哪个更大权重

    💡 一句话记忆法：卡尔曼滤波 = "先预测、再修正"的递推加权平均，K 越大越信测量！

    学这个知识点能解决什么： - GPS/INS 组合导航中的位置和速度估计 - 雷达目标跟踪中的运动状态估计 - 传感器融合：综合多个不完美传感器的测量

    3. 严谨核心¶

    数学对象¶

    卡尔曼滤波（Kalman Filter）是一种递推最优状态估计算法，适用于线性动态系统的状态估计。与维纳滤波不同，卡尔曼滤波基于状态空间模型，不需要信号是平稳的，且采用递推计算，适合实时处理。

    基本假设¶

    • 系统为线性状态空间模型
    • 过程噪声和观测噪声均为零均值高斯白噪声
    • 在线性高斯假设下，卡尔曼滤波器是 MMSE 意义下的最优估计

    核心公式¶

    状态空间模型：

    \[\mathbf{x}[n] = \mathbf{F}\mathbf{x}[n-1] + \mathbf{G}\mathbf{w}[n]\]
    \[\mathbf{y}[n] = \mathbf{H}\mathbf{x}[n] + \mathbf{v}[n]\]

    其中 \(\mathbf{F}\) 为状态转移矩阵，\(\mathbf{H}\) 为观测矩阵，\(\mathbf{w}[n]\) 为过程噪声，\(\mathbf{v}[n]\) 为观测噪声，\(E[\mathbf{w}\mathbf{w}^H]=\mathbf{Q}\)，\(E[\mathbf{v}\mathbf{v}^H]=\mathbf{R}\)。

    预测步骤：

    \[\hat{\mathbf{x}}[n|n-1] = \mathbf{F}\hat{\mathbf{x}}[n-1|n-1]\]
    \[\mathbf{P}[n|n-1] = \mathbf{F}\mathbf{P}[n-1|n-1]\mathbf{F}^H + \mathbf{G}\mathbf{Q}\mathbf{G}^H\]

    更新步骤：

    \[\mathbf{K}[n] = \mathbf{P}[n|n-1]\mathbf{H}^H \left(\mathbf{H}\mathbf{P}[n|n-1]\mathbf{H}^H + \mathbf{R}\right)^{-1}\]
    \[\hat{\mathbf{x}}[n|n] = \hat{\mathbf{x}}[n|n-1] + \mathbf{K}[n]\left(\mathbf{y}[n] - \mathbf{H}\hat{\mathbf{x}}[n|n-1]\right)\]
    \[\mathbf{P}[n|n] = \left(\mathbf{I} - \mathbf{K}[n]\mathbf{H}\right)\mathbf{P}[n|n-1]\]

    其中 \(\mathbf{K}[n]\) 为卡尔曼增益，\(\mathbf{P}\) 为估计误差协方差矩阵。

    卡尔曼滤波变体对比¶

    变体 系统模型 线性化方式 计算复杂度 精度
    标准卡尔曼滤波 (KF) 线性高斯 无需线性化 O(n³) 最优 (线性下)
    扩展卡尔曼滤波 (EKF) 非线性高斯 一阶泰勒展开 (雅可比矩阵) O(n³) 一阶近似，可能发散
    无迹卡尔曼滤波 (UKF) 非线性高斯 Sigma点采样 O(n³) 二阶精度，无需求导
    粒子滤波 (PF) 非线性非高斯 蒙特卡洛采样 O(N·n) 通用但计算量大

    适用条件和边界¶

    • 标准 KF 要求线性系统 + 高斯噪声；非线性系统需要 EKF/UKF
    • 卡尔曼增益 K 由 P 和 H 自动计算，会随时间自适应变化
    • 初始协方差影响收敛速度：设太大则前期过度依赖测量，设太小则可能忽略新观测导致发散
    • 模型失配、噪声统计不准或数值误差都可能导致滤波器发散

    4. 方法流程¶

    输入： - 状态空间模型参数：\(\mathbf{F}\)（状态转移）、\(\mathbf{H}\)（观测矩阵）、\(\mathbf{Q}\)（过程噪声协方差）、\(\mathbf{R}\)（观测噪声协方差） - 观测序列 \(\mathbf{y}[1], \mathbf{y}[2], \ldots\) - 初始状态估计 \(\hat{\mathbf{x}}[0|0]\) 和初始协方差 \(\mathbf{P}[0|0]\)

    处理步骤： 1. 初始化：设定 \(\hat{\mathbf{x}}[0|0]\) 和 \(\mathbf{P}[0|0]\) 2. 预测步：用状态方程预测下一时刻的状态和协方差 3. 更新步：计算卡尔曼增益 \(\mathbf{K}\)，融合预测与新观测 4. 重复步骤 2-3，逐时刻递推

    输出解释： - \(\hat{\mathbf{x}}[n|n]\)：当前时刻的最优状态估计 - \(\mathbf{P}[n|n]\)：估计的不确定性（对角线元素为各状态分量的方差） - \(\mathbf{K}[n]\)：卡尔曼增益，大则更信测量，小则更信模型

    常见参数选择： - \(\mathbf{Q}\) 大 → 模型不确定性高 → 更信测量（K 增大） - \(\mathbf{R}\) 大 → 测量噪声大 → 更信模型（K 减小） - \(\mathbf{P}[0|0]\) 大 → 初始不确定 → 快速收敛到观测值

    5. Python最小实验¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib"])

    # === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

# 一维卡尔曼滤波：跟踪匀速运动目标
np.random.seed(42)
N = 50

# 真实轨迹：匀速运动（位置从0开始，每步+1）
true_pos = np.arange(N, dtype=float)

# 含噪观测
R_meas = 4.0
z = true_pos + np.random.randn(N) * np.sqrt(R_meas)

# 状态空间模型：状态=[位置, 速度]
F = np.array([[1, 1], [0, 1]])       # 状态转移矩阵
H = np.array([[1, 0]])               # 观测矩阵
Q = np.array([[0.1, 0], [0, 0.1]])   # 过程噪声协方差
R = np.array([[R_meas]])             # 观测噪声协方差

# 卡尔曼滤波递推
x_est = np.array([[0.0], [0.0]])     # 初始状态估计
P = np.eye(2) * 10                   # 初始协方差（较大表示不确定）
est_pos = np.zeros(N)
K_gain = np.zeros(N)

for i in range(N):
    # 预测步
    x_pred = F @ x_est
    P_pred = F @ P @ F.T + Q
    # 更新步
    K = P_pred @ H.T @ np.linalg.inv(H @ P_pred @ H.T + R)
    x_est = x_pred + K @ (z[i] - H @ x_pred)
    P = (np.eye(2) - K @ H) @ P_pred
    est_pos[i] = x_est[0, 0]
    K_gain[i] = K[0, 0]

fig, axes = plt.subplots(2, 1, figsize=(10, 6))
axes[0].plot(true_pos, 'b-', label='真实轨迹', linewidth=2)
axes[0].plot(z, 'rx', alpha=0.5, label='Noisy Obs')
axes[0].plot(est_pos, 'g-', label='卡尔曼滤波估计', linewidth=2)
axes[0].set_title('一维卡尔曼滤波：匀速运动目标跟踪')
axes[0].set_xlabel('Time Step')
axes[0].set_ylabel('位置')
axes[0].legend()

axes[1].plot(K_gain, 'r-')
axes[1].set_title('卡尔曼增益 K（逐渐收敛→滤波器自动平衡模型与测量）')
axes[1].set_xlabel('Time Step')
axes[1].set_ylabel('Gain K')
axes[1].grid(True)

plt.tight_layout()
plt.show()

    结果解释： - 上图：卡尔曼滤波估计（绿线）比含噪观测（红叉）更接近真实轨迹（蓝线），有效去除了观测噪声 - 下图：卡尔曼增益 K 从初始的较大值逐渐收敛到稳定值——说明滤波器从"主要信观测"过渡到"平衡模型与测量"

    6. 常见误区¶

    易错点 正确理解
    卡尔曼滤波只能用于线性系统 标准 KF 要求线性，但 EKF（线性化）和 UKF（sigma 点）可处理非线性系统
    卡尔曼增益 K 需要手动设定 K 由 P 和 H 自动计算，会随时间自适应变化——模型可信时 K 变小，测量可信时 K 变大
    初始协方差不重要 初始值影响收敛速度：设太大则前期过度依赖测量，设太小则可能忽略新观测导致发散
    卡尔曼滤波一定收敛 模型失配、噪声统计不准或数值误差都可能导致滤波器发散

    7. 启发问题与创新拓展¶

    一个反例或边界问题： 如果过程噪声 \(\mathbf{Q}\) 设得太小（过度信任模型），而实际系统有未建模的动态（如突然转弯），卡尔曼滤波会"固执"地按模型预测，导致跟踪失败。这就是为什么实际系统中常需要"自适应卡尔曼滤波"来在线调整 \(\mathbf{Q}\) 和 \(\mathbf{R}\)。

    一个跨学科迁移场景： 自动驾驶中，GPS + IMU + 视觉里程计的多传感器融合，本质是扩展卡尔曼滤波的状态估计问题。GPS 提供绝对位置（大 R），IMU 提供高频运动信息（小 R 但漂移），卡尔曼滤波自动融合两者。

    一个可做成课程项目的小想法： 实现一个"卡尔曼滤波 vs 滑动平均"对比实验：对同一段含噪信号，分别用卡尔曼滤波和简单滑动平均去噪，比较收敛速度、稳态误差和对突变的响应能力。
