    经典信号处理¶

    [!tip] 🎯 这个系列在讲什么？ 一句话概括：经典信号处理教你如何把一个随时间变化的物理量（信号）"看懂"——拆开看、滤干净、算出来。

    如果你学过电路、通信、音频处理、图像处理中的任何一门课，你就已经用过经典信号处理的工具了。这个系列把最核心的5个知识点串起来，形成一条完整的认知链路。

    2. 通俗入口¶

    一个真实问题¶

    ![[经典信号处理-流程图_ai.png|地铁语音通话的信号处理全流程]]

    你在嘈杂的地铁里用手机语音通话。对方的声音很清楚，背景噪音被压低了。这件"理所当然"的事，背后发生了什么？

    1. 你的手机麦克风采集声音 → 这是采样（[[采样与重建]]）
    2. 芯片把声波变成数字序列 → 这是信号的数字化表示（[[信号与系统基础]]）
    3. 算法分析声音里哪些是语音、哪些是噪音 → 这是频谱分析（[[傅里叶分析]]）
    4. 一个"筛子"滤掉噪音、保留语音 → 这是数字滤波器（[[数字滤波器设计]]）
    5. 整个过程用差分方程高效计算 → 这是Z变换的用武之地（[[Z变换]]）

    这5步，就是经典信号处理的完整链条。

    学这个能解决什么？¶

    场景 用到的知识点 具体做什么
    🎵 音频降噪 傅里叶分析 + 滤波器 识别噪音频率并滤除
    📱 语音通话 采样 + 滤波 + Z变换 采集-编码-传输-解码-播放
    🏥 医学影像（CT/MRI） 傅里叶分析 从投影数据重建图像
    📡 通信系统 采样 + 滤波 + 调制 信号的调制解调和信道滤波
    🖼️ 图像处理 傅里叶分析 + 滤波 图像增强、去噪、压缩
    🎸 音乐均衡器 傅里叶分析 + 滤波器 调节不同频段的音量
    📊 数据压缩（JPEG/MP3） 傅里叶分析 去掉人耳/眼不敏感的频率分量
    🔭 天文信号处理 采样 + 频谱分析 从微弱信号中提取天体信息

    3. 严谨核心¶

    为什么叫"经典"？¶

    因为这些理论在1940-1980年代就已经成熟，是所有现代信号处理技术的地基。今天你听到的深度学习语音识别、5G通信、自动驾驶雷达——底层仍然在用这些"经典"工具做预处理。

    对比 经典信号处理 现代/深度学习方法
    核心思想 数学推导，解析解 数据驱动，端到端学习
    优势 可解释、理论保证、低计算量 自适应、处理非线性问题
    典型应用 滤波、频谱分析、采样 语音识别、图像生成
    关系 地基 地基之上的建筑

    [!important] 💡 学习建议 不要把经典方法和深度学习对立起来。理解经典信号处理，才能知道深度学习在信号领域"替代了什么、没替代什么"。 比如，深度学习语音识别的前端仍然用MFCC特征——而MFCC的基础就是傅里叶分析和滤波器组。

    5个知识点的逻辑关系¶

    ![[经典信号处理-知识图谱_ai.png|经典信号处理知识点逻辑关系图]]

    各子知识点核心公式概览¶

    信号与系统基础：LTI系统卷积 $\(y[n] = \sum_{k=-\infty}^{\infty} h[k] \cdot x[n-k] = (h * x)[n]\)$

    采样与重建（奈奎斯特定理）： $\(f_s \geq 2 f_{\max}\)$ 采样率必须大于信号最高频率的2倍，否则发生混叠。

    傅里叶分析（DFT）： $\(X[k] = \sum_{n=0}^{N-1} x[n] e^{-j2\pi kn/N}, \quad k = 0, 1, \ldots, N-1\)$

    数字滤波器（FIR/IIR）： - FIR：\(y[n] = \sum_{k=0}^{M} b_k x[n-k]\) （有限脉冲响应） - IIR：\(y[n] = \sum_{k=0}^{M} b_k x[n-k] - \sum_{k=1}^{N} a_k y[n-k]\) （无限脉冲响应）

    Z变换： $\(X(z) = \sum_{n=-\infty}^{\infty} x[n] z^{-n}\)$ 系统函数：\(H(z) = \frac{Y(z)}{X(z)} = \frac{\sum_{k=0}^{M} b_k z^{-k}}{1 + \sum_{k=1}^{N} a_k z^{-k}}\)

    适用条件与边界¶

    • 奈奎斯特定理要求信号是带限的（有最高频率）
    • DFT假设信号在一个窗口内是周期性的
    • FIR滤波器总是稳定的；IIR滤波器需要所有极点在单位圆内
    • 经典方法假设信号是确定性的或平稳的；非平稳/随机信号需要统计方法

    4. 方法流程¶

    经典信号处理全流程¶

    以地铁语音通话降噪为例：

    输入：含噪语音信号 \(x[n]\)（语音 + 背景噪声），采样率 \(f_s = 8000\) Hz

    步骤：

    1. 采样（[[采样与重建]]）：麦克风将连续声波转换为离散数字序列
    2. 频谱分析（[[傅里叶分析]]）：用FFT计算信号频谱，识别语音频段（300-3400Hz）和噪声频段
    3. 滤波（[[数字滤波器设计]]）：设计带通滤波器，保留语音频段，滤除噪声
    4. 系统分析（[[Z变换]]）：用系统函数分析滤波器的频率响应和稳定性
    5. 重建：IFFT或直接输出滤波后的信号

    输出：降噪后的语音信号 \(\hat{x}[n]\)

    参数选择指南¶

    环节 关键参数 选择建议
    采样 采样率 \(f_s\) 电话语音 8kHz，高保真 44.1kHz
    FFT 窗口长度 \(N\) 语音分析通常 20-40ms（160-320点@8kHz）
    滤波器 截止频率、阶数 语音保留 300-3400Hz；阶数越高过渡带越窄
    Z变换 极点位置 IIR滤波器极点必须在单位圆内

    5. Python 最小实验¶

    数据生成¶

    模拟地铁语音通话的降噪全流程：采样 → 频谱分析 → 滤波 → 重建。

    关键函数¶

    • np.fft.fft() / np.fft.ifft() — FFT频谱分析与逆变换重建
    • np.fft.fftfreq() — 频率轴计算
    • 频域滤波（将特定频率分量置零）

    代码与曲线¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib"])

    # === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

# 1. 采样：模拟麦克风采集声音（语音 + 噪声）
fs = 8000  # 采样率 8000 Hz（电话语音标准）
t = np.arange(0, 0.5, 1/fs)  # 0.5秒

# 语音信号：300Hz 基频 + 谐波（模拟人声）
voice = 0.5*np.sin(2*np.pi*300*t) + 0.3*np.sin(2*np.pi*600*t) + 0.2*np.sin(2*np.pi*900*t)

# 噪声：高频随机噪声（模拟地铁背景噪音）
np.random.seed(42)
noise = 0.4 * np.random.randn(len(t))

# 混合信号（麦克风采集到的）
x = voice + noise

# 2. 频谱分析：FFT 看看信号里有什么频率
N = len(x)
X = np.fft.fft(x)
freqs = np.fft.fftfreq(N, 1/fs)

# 只看正频率
positive_mask = freqs > 0
freqs_pos = freqs[positive_mask]
X_pos = np.abs(X[positive_mask]) * 2 / N

# 3. 滤波：设计低通滤波器，保留语音（300-1000Hz），滤除高频噪声
# 简单的频域滤波：将高于1500Hz的分量置零
cutoff = 1500  # 截止频率
X_filtered = X.copy()
X_filtered[np.abs(freqs) > cutoff] = 0

# 4. 逆FFT重建滤波后的信号
x_filtered = np.real(np.fft.ifft(X_filtered))

# 可视化全流程
fig, axes = plt.subplots(2, 2, figsize=(12, 8))

# 原始时域信号
axes[0, 0].plot(t[:200], x[:200], 'steelblue')
axes[0, 0].set_xlabel('Time (s)')
axes[0, 0].set_ylabel('Amplitude')
axes[0, 0].set_title('① 采样：麦克风采集的含噪语音')
axes[0, 0].grid(True)

# 频谱分析
axes[0, 1].plot(freqs_pos[:500], X_pos[:500], 'coral')
axes[0, 1].axvline(x=1500, color='green', linestyle='--', label='截止频率 1500Hz')
axes[0, 1].set_xlabel('Frequency (Hz)')
axes[0, 1].set_ylabel('Amplitude')
axes[0, 1].set_title('② 频谱分析：FFT 频谱')
axes[0, 1].legend()
axes[0, 1].grid(True)

# 滤波后的频谱
X_filtered_pos = np.abs(X_filtered[positive_mask]) * 2 / N
axes[1, 0].plot(freqs_pos[:500], X_filtered_pos[:500], 'seagreen')
axes[1, 0].set_xlabel('Frequency (Hz)')
axes[1, 0].set_ylabel('Amplitude')
axes[1, 0].set_title('③ 滤波：低通滤波后的频谱')
axes[1, 0].grid(True)

# 重建信号
axes[1, 1].plot(t[:200], x_filtered[:200], 'seagreen', label='Filtered')
axes[1, 1].plot(t[:200], voice[:200], 'coral', alpha=0.5, label='Original Speech')
axes[1, 1].set_xlabel('Time (s)')
axes[1, 1].set_ylabel('Amplitude')
axes[1, 1].set_title('④ 重建：滤波后的语音信号')
axes[1, 1].legend()
axes[1, 1].grid(True)

plt.tight_layout()
plt.show()

# 计算信噪比改善
snr_before = 10 * np.log10(np.mean(voice**2) / np.mean(noise**2))
residual_noise = x_filtered - voice
snr_after = 10 * np.log10(np.mean(voice**2) / np.mean(residual_noise**2))
print(f"滤波前信噪比: {snr_before:.1f} dB")
print(f"滤波后信噪比: {snr_after:.1f} dB")
print(f"信噪比改善: {snr_after - snr_before:.1f} dB")

    结果解释¶

    运行结果解读：这个示例展示了经典信号处理的完整链条——采样（模拟麦克风）→ 频谱分析（FFT）→ 滤波（低通滤波器）→ 重建（IFFT）。频谱图上可以看到语音的谐波成分（300/600/900Hz）和高频噪声，低通滤波后高频噪声被滤除，信噪比显著提升。这就是地铁语音通话降噪的基本原理。

    • ① 采样：时域波形中语音和噪声混叠，肉眼难以区分
    • ② 频谱分析：FFT揭示了信号的频率成分——300/600/900Hz的语音谐波清晰可见，高频区域有噪声能量
    • ③ 滤波：截止频率1500Hz的低通滤波器将高频噪声分量置零
    • ④ 重建：IFFT重建后的信号波形更干净，接近原始语音

    6. 常见误区¶

    公式误解¶

    • ❌ "奈奎斯特定理说采样率等于2倍最高频率就够了" — 理论上 \(f_s = 2f_{\max}\) 刚好不混叠，但实际中需要留余量（通常 \(f_s \geq 2.5 \sim 5 \times f_{\max}\)），且需要抗混叠滤波器配合
    • ❌ "FFT的频率分辨率是 \(f_s/N\)" — 这只是频率轴的采样间隔，真正的频率分辨率由信号的有效观测时长决定：\(\Delta f \approx 1/T\)

    单位误解¶

    • ❌ "DFT的结果 \(X[k]\) 的单位是Hz" — \(X[k]\) 的单位与 \(x[n]\) 相同（如电压V），它是复数振幅；\(|X[k]|^2\) 才与功率相关
    • ❌ "dB是绝对单位" — dB是相对比值，需要一个参考基准（如 dBV 以1V为参考，dBFS 以满量程为参考）

    参数误解¶

    • ❌ "FFT点数越大越好" — 增加FFT点数（补零）只增加频率采样密度，不提高分辨率。真正的分辨率提升需要更长的观测数据
    • ❌ "滤波器阶数越高越好" — 阶数越高过渡带越窄，但延迟越大、计算量越高，且可能引入数值不稳定

    适用条件误解¶

    • ❌ "对任何信号都能用DFT做频谱分析" — DFT隐含周期延拓假设，对非周期信号会产生频谱泄漏，需要加窗处理
    • ❌ "IIR滤波器比FIR好因为效率高" — IIR效率高但可能不稳定、有非线性相位；FIR一定稳定且可实现线性相位，各有适用场景
    • ❌ "经典方法过时了，深度学习可以替代一切" — 深度学习的音频前端（MFCC特征）仍然依赖傅里叶分析和滤波器组，经典方法是不可或缺的基础

    7. 启发问题与创新拓展¶

    一个反例或边界问题¶

    欠采样的"惊喜"：根据奈奎斯特定理，对一个7Hz的正弦波以10Hz采样（\(f_s < 2f_{\max}\)），理论上应该发生混叠。但如果信号恰好只有7Hz这一个频率，采样后会出现一个3Hz的"假信号"——这就是混叠。然而，如果你事先知道信号只包含7Hz这一个频率，用参数化方法（如拟合正弦模型）仍然可以精确恢复。这说明奈奎斯特定理是一个充分条件而非必要条件。

    一个跨学科迁移场景¶

    天文观测中的经典信号处理：射电望远镜接收到的宇宙信号极其微弱，需要在强噪声中提取微弱的天体信号。这和地铁降噪的原理完全一致：采样 → 频谱分析 → 窄带滤波 → 信号检测。脉冲星的发现（1967年）正是通过分析信号的周期性频谱特征实现的。

    一个可做成课程项目的小想法¶

    "手机变示波器"：用手机的麦克风采集声音信号（如音叉、吉他弦），在Python中实时计算FFT频谱并显示，实现一个简易的"频谱分析仪"。进阶版可以加入数字滤波器，实现实时降噪。这个项目串联了采样、FFT、滤波器设计的全部核心知识点。
