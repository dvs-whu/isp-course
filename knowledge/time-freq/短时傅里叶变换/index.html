    2. 通俗入口¶

    想象你在一个嘈杂的餐厅里录了一段音频，你想知道"什么时候出现了什么频率的声音"。普通的傅里叶变换只能告诉你"整段录音里有哪些频率"，但无法告诉你"这些频率出现在什么时间"。

    STFT 的解决方案很直觉： - 拿一个滑动窗口（比如 25ms 长）沿着时间轴移动 - 在每个位置对窗口内的信号片段做 FFT - 把所有位置的频谱拼在一起，得到一个时间-频率的二维图

    这就是频谱图（Spectrogram）的由来——横轴是时间，纵轴是频率，颜色表示能量强度。

    窗长的两难困境： - 窗越长 → 频率分辨率越高（能区分相近频率），但时间分辨率越差（不知道频率变化的确切时刻） - 窗越短 → 时间分辨率越高，但频率分辨率越差 - 这就是 Heisenberg 不确定性原理在信号分析中的体现

    生活类比：STFT 就像"拿手电筒沿时间轴扫信号，每个位置照出一幅频谱"。手电筒的光圈大小就是窗长——光圈越大看到的频率细节越多，但你不确定这些细节出现在光圈内的哪个位置。

    💡 一句话记忆法：STFT = 滑动窗口 + FFT = 一维信号的"时间-频率"二维展开。

    你需要重点掌握的： 1. STFT 的本质：滑动窗口 + FFT，把一维信号切成重叠小段逐段做频谱分析 2. 窗长决定分辨率权衡：窗越长→频率分辨率越高但时间分辨率越差，窗越短则反之（不确定性原理） 3. 频谱图（Spectrogram）= STFT 模的平方，横轴时间、纵轴频率、颜色表示能量强度

    作为进阶了解的（期末不要求推导）： - 不同窗函数（汉宁窗、汉明窗等）对频谱泄漏的影响及理论分析

    3. 严谨核心¶

    定义与概念¶

    短时傅里叶变换（Short-Time Fourier Transform, STFT）是时频分析中最基本的方法之一。其核心思想是：在信号上滑动一个有限长度的窗函数，对加窗后的局部信号片段进行傅里叶变换，从而获得信号在不同时间点附近的频率信息。

    STFT 将一维时域信号映射为二维时频表示 \(S(t, f)\)，使我们能够同时观察信号的时间特征和频率特征。窗函数的选择（形状与宽度）直接影响时频分辨率：窗越窄，时间分辨率越高，频率分辨率越低；窗越宽则反之。这一矛盾关系即为 Heisenberg 不确定性原理在信号分析中的体现。

    STFT 的一个关键性质是其完全可逆性：从时频表示可以无损地恢复原始信号。

    核心公式¶

    连续 STFT 定义：

    \[ S(t, f) = \int_{-\infty}^{+\infty} x(\tau)\, w(\tau - t)\, e^{-j2\pi f \tau}\, d\tau \]

    其中 \(x(t)\) 为输入信号，\(w(t)\) 为窗函数。

    逆 STFT（信号重建）：

    \[ x(t) = \int_{-\infty}^{+\infty} \int_{-\infty}^{+\infty} S(\tau, f)\, w(t - \tau)\, e^{j2\pi f t}\, df\, d\tau \]

    离散 STFT：

    \[ S(m, k) = \sum_{n=0}^{N-1} x(n + mH)\, w(n)\, e^{-j2\pi kn/N} \]

    其中 \(H\) 为帧移（hop size），\(N\) 为窗长，\(m\) 为帧索引，\(k\) 为频率索引。该式可通过 FFT 高效计算。

    时间-频率分辨率关系（不确定性原理）：

    \[ \Delta t \cdot \Delta f \geq \frac{1}{4\pi} \]

    STFT 使用固定窗长，因此时频分辨率在整个时频平面上是恒定的。

    STFT 计算流程¶

    常用窗函数对比¶

    窗函数 主瓣宽度 旁瓣衰减 频率分辨率 适用场景
    矩形窗 窄 差（-13dB） 高 瞬态信号、短时分析
    汉宁窗 中 好（-31dB） 中 通用频谱分析
    汉明窗 中 较好（-43dB） 中 语音信号处理
    布莱克曼窗 宽 优秀（-58dB） 低 需要高动态范围的场景
    高斯窗 可调 可调 可调 时频分析、STFT 默认选择

    典型应用¶

    • 语音信号处理：语音识别、说话人识别中提取 MFCC 等特征的标准前端处理
    • 音频分析与合成：音乐信号的频谱图（spectrogram）分析、音高检测、音频时间拉伸
    • 雷达与声纳：目标回波信号的时频特征提取
    • 振动分析：机械故障诊断中的转频及谐波跟踪
    • 通信系统：OFDM 信号的调制与解调本质上可看作 STFT 的逆过程
    • 地震信号处理：地震波的频谱分析

    4. 方法流程¶

    输入：一维时间信号 \(x[n]\)，采样率 \(f_s\)

    Step 1：分帧加窗 - 将信号分成重叠的短帧（典型帧长 20~40ms，帧移为帧长的 50%~75%） - 每帧乘以窗函数 \(w[n]\)（如汉宁窗） - 第 \(m\) 帧信号：\(x_m[n] = x[n + mH] \cdot w[n]\)

    Step 2：FFT 变换 - 对每帧做 \(N\) 点 FFT（\(N\) 通常取 2 的幂次） - 得到复数频谱 \(X_m[k]\)

    Step 3：计算频谱图 - 功率谱图：\(|X_m[k]|^2\) - 对数谱图（dB）：\(10\log_{10}|X_m[k]|^2\) - 横轴为帧索引（时间），纵轴为频率索引，颜色为能量

    Step 4：后处理（可选） - MFCC 提取：对功率谱做 Mel 滤波 → 取对数 → DCT - 声谱图增强：如 Wiener 滤波在时频域的语音增强

    参数选择建议： - 帧长：语音通常 25ms（如 16kHz 采样率下 400 点），音乐通常 46ms（2048 点） - 帧移：帧长的 50%（50% 重叠）是最常见的选择 - FFT 点数：\(\geq\) 帧长，通常取 2 的幂次（512、1024、2048） - 窗函数：通用选择汉宁窗；需要高动态范围选布莱克曼窗

    5. Python 最小实验¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib scipy - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib", "scipy"])

    # === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
from scipy import signal
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

# 生成线性调频信号（chirp）：频率从 10Hz 线性增长到 200Hz
fs = 1000
t = np.linspace(0, 2, 2*fs)
x = signal.chirp(t, 10, t[-1], 200, method='linear')
x += 0.3 * np.random.randn(len(x))  # 添加少量噪声

# 计算 STFT 频谱图（不同窗长对比）
fig, axes = plt.subplots(3, 1, figsize=(10, 10))
# 上图：时域波形
axes[0].plot(t, x, 'b-', linewidth=0.5)
axes[0].set_title('线性调频信号（Chirp）时域波形')
axes[0].set_xlabel('Time (s)')
axes[0].set_ylabel('幅值')

# 中图：短窗（高时间分辨率）
f1, t1, Sxx1 = signal.spectrogram(x, fs=fs, nperseg=64, noverlap=48)
im1 = axes[1].pcolormesh(t1, f1, 10*np.log10(Sxx1+1e-10), shading='gouraud', cmap='jet')
axes[1].set_title('STFT 频谱图 - 短窗（nperseg=64）：时间分辨率高，频率分辨率低')
axes[1].set_xlabel('Time (s)')
axes[1].set_ylabel('Frequency (Hz)')
plt.colorbar(im1, ax=axes[1], label='Power (dB)')

# 下图：长窗（高频率分辨率）
f2, t2, Sxx2 = signal.spectrogram(x, fs=fs, nperseg=256, noverlap=224)
im2 = axes[2].pcolormesh(t2, f2, 10*np.log10(Sxx2+1e-10), shading='gouraud', cmap='jet')
axes[2].set_title('STFT 频谱图 - 长窗（nperseg=256）：频率分辨率高，时间分辨率低')
axes[2].set_xlabel('Time (s)')
axes[2].set_ylabel('Frequency (Hz)')
plt.colorbar(im2, ax=axes[2], label='Power (dB)')

plt.suptitle('STFT 窗长对时频分辨率的影响', fontsize=15)
plt.tight_layout()
plt.show()

    结果解释：三张图分别展示了原始 chirp 信号、短窗 STFT 和长窗 STFT。短窗（64 点）能清晰追踪频率随时间的变化轨迹，但频谱较"糊"；长窗（256 点）频率分辨率更高，能看到更精细的频率成分，但时间上变"模糊"了。这直观地展示了不确定性原理的约束：时间分辨率和频率分辨率不可兼得。

    6. 常见误区¶

    误区 正确理解
    STFT 的时频分辨率可以同时提高 受不确定性原理限制，\(\Delta t \cdot \Delta f \geq \frac{1}{4\pi}\)，提高一个必须牺牲另一个
    窗函数的选择不重要 窗函数影响频谱泄漏和主瓣宽度；矩形窗旁瓣高（-13dB），汉宁窗旁瓣低（-31dB），选择不当会导致虚假频率成分
    STFT 频谱图的纵轴就是频率 频率分辨率 \(\Delta f = f_s / N\)，\(N\) 是 FFT 点数；频率轴的精度受限于窗长
    帧移越小越好 帧移太小导致计算量大增且冗余信息多；50% 重叠（帧移=帧长/2）通常足够
    STFT 只适用于语音信号 STFT 适用于任何需要时频分析的信号：振动、雷达、地震、音乐等
    频谱图中的颜色表示"某个频率的幅值" 严格来说是功率谱密度（$
    STFT 不能处理非平稳信号 STFT 正是为了分析非平稳信号而设计的；但它假设信号在窗内是近似平稳的，这对快速变化的信号可能不成立

    7. 启发问题与创新拓展¶

    思考题¶

    1. 不确定性原理 \(\Delta t \cdot \Delta f \geq \frac{1}{4\pi}\) 的物理含义是什么？为什么 STFT 在整个时频平面上使用固定的分辨率，而小波变换可以自适应？
    2. 如果信号包含一个频率突变（如从 100Hz 瞬间跳到 200Hz），STFT 的窗长如何影响我们检测这个突变的能力？
    3. OFDM（正交频分复用）为什么可以看作 STFT 的逆过程？这对理解 5G/Wi-Fi 通信有什么帮助？
    4. 频谱图中的"频谱泄漏"是怎么产生的？如何通过窗函数选择来缓解？

    创新拓展方向¶

    • 多窗谱分析（Multitaper）：使用多个正交窗函数取平均，降低频谱估计方差
    • S 变换：结合 STFT 和小波变换的优点，窗长随频率自适应变化
    • STFT 与深度学习：音频领域的神经网络通常以频谱图（STFT 输出）作为输入特征
    • 实时 STFT：嵌入式系统中的滑动 DFT（Sliding DFT）算法，逐样本更新频谱
    • 相位信息的利用：Griffin-Lim 算法仅从幅度谱重建信号，相位检索是活跃研究方向
    • 时频域信号处理：在 STFT 域进行语音增强、源分离等处理
