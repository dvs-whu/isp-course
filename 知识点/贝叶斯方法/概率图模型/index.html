    概率图模型¶

    1. 学习定位¶

    📚 拓展知识点说明

    定位：本知识点是本科课程的拓展内容，适合学有余力的同学深入学习。 一句话理解：用图结构描述随机变量之间的依赖关系。 前置要求：贝叶斯推断、图论基础 与本科课程的关系：卡尔曼滤波是线性高斯图模型

    前置知识： - [[贝叶斯推断]]：条件概率、贝叶斯网络的推断基础 - 图论基础：图、树、团、DAG 等基本概念

    与本科课程的关联： - [[../统计信号处理/卡尔曼滤波]] — 卡尔曼滤波可视为线性高斯状态空间模型（一种特殊的概率图模型） - [[../语音信号处理/语音增强]] — 贝叶斯语音增强方法（如 MMSE 估计器）依赖图模型中的条件推断

    相关知识点： - [[贝叶斯推断]] — 概率推断 - [[../图信号处理/图论基础]] — 图结构 - [[MCMC]] — 图模型采样 - [[变分推断]] — 图模型推断 - [[../深度学习/深度学习基础]] — 深度生成模型

    2. 通俗入口¶

    想象你是一个侦探，面对一起交通事故。你知道下雨会让路面变湿，路面变湿会导致交通拥堵。但你只能观察到"交通拥堵"这个结果，想反过来推断"是否下雨"。

    在脑中画一张因果链：下雨 → 路湿 → 拥堵。这张图就是概率图模型的雏形——用节点表示变量（下雨、路湿、拥堵），用箭头表示因果关系，每个节点附带一个条件概率表。

    概率图模型的核心思想： - 图结构：用图直观地表达"谁影响谁" - 条件独立性：知道中间变量后，某些变量就不再相关（知道路湿了，是否下雨对拥堵的判断就不再有额外信息） - 推断：给定某些已知变量，计算其他变量的后验概率

    为什么需要概率图模型？ 现实世界中变量之间的关系复杂且不确定。直接建模所有变量的联合分布 \(p(x_1, x_2, \ldots, x_n)\) 需要指数级的参数。概率图模型利用条件独立性，将联合分布分解为多个局部条件概率的乘积，大大减少了参数量。

    生活类比：

    类比对象 概率图模型对应 说明
    家族族谱 有向图（贝叶斯网络） 箭头表示"谁生了谁"，上一代影响下一代
    朋友圈关系 无向图（马尔可夫随机场） 没有方向，只有"谁和谁相关"
    通过中间人传递消息 信念传播算法 信息沿着图的边传递，直到收敛

    💡 一句话记忆法：概率图模型 = 概率论 + 图论，用图的结构来"压缩"复杂概率分布的描述。

    3. 严谨核心¶

    定义与概念¶

    概率图模型（Probabilistic Graphical Models, PGM）是用图结构表示多元概率分布中条件独立性的框架。图中的节点对应随机变量，边编码变量间的依赖关系。

    两大类： - 有向图模型（贝叶斯网络）：用有向无环图（DAG）表示因果/生成关系，联合分布分解为条件概率的乘积 - 无向图模型（马尔可夫随机场）：用无向图表示对称相关关系，联合分布由势函数定义

    关键概念： - D-分离（d-separation）：判断有向图中条件独立性 - 团与最大团：无向图中的全连接子图 - 消息传递算法：信念传播（Belief Propagation）、和积算法

    核心公式¶

    贝叶斯网络的联合分布分解：

    \[ p(x_1, \ldots, x_n) = \prod_{i=1}^{n} p(x_i \mid \text{pa}(x_i)) \]

    其中 \(\text{pa}(x_i)\) 为节点 \(x_i\) 的父节点集合。

    马尔可夫随机场的联合分布：

    \[ p(x_1,\ldots,x_n) = \frac{1}{Z}\prod_{c \in \mathcal{C}} \psi_c(x_c) \]

    其中 \(\psi_c\) 为定义在团 \(c\) 上的势函数，\(Z\) 为配分函数。

    信念传播（和积消息传递）：从节点 \(i\) 到 \(j\) 的消息：

    \[ m_{i\to j}(x_j) = \sum_{x_i} \psi(x_i)\,\psi(x_i,x_j)\,\prod_{k \in \mathcal{N}(i)\setminus j} m_{k\to i}(x_i) \]

    概率图模型对比表¶

    模型 图类型 联合分布表示 推断算法 学习方法 典型应用 优点 缺点
    贝叶斯网络 有向DAG \(\prod_i p(x_i \mid pa(x_i))\) 变量消元、BP MLE/EM、贝叶斯 因果推断、诊断 可表达因果关系 需确定DAG结构
    马尔可夫随机场 无向图 (1/Z)Π ψc(xc) BP、MCMC、图割 伪似然、对比散度 图像分割、空间模型 对称依赖 配分函数难算
    因子图 二部图 Π fi(xi) 和积/最大积 消息传递框架 信道编码、SP 通用框架 图结构影响效率
    HMM 链式有向 发射+转移概率 前后向算法、Viterbi Baum-Welch(EM) 语音、序列标注 高效精确推断 条件独立假设强
    CRF 链式/树无向 条件概率 BP、Viterbi 最大似然 NLP、序列标注 无需建模输入分布 训练复杂
    LDA 有向(生成) 主题-文档-词 Gibbs采样、VI 变分EM、Collapsed Gibbs 文本主题建模 无监督主题发现 主题数需预设

    典型应用¶

    • 语音识别中的隐马尔可夫模型（HMM）
    • 图像分割中的马尔可夫随机场
    • 信道编译码中的因子图与 LDPC 码（Kschischang et al., 2001）
    • 传感器网络中的分布式推断
    • 贝叶斯网络用于因果推断与故障诊断

    4. 方法流程¶

    贝叶斯网络推断流程¶

    Step 1：建模 - 确定随机变量及其取值范围 - 根据领域知识或数据学习 DAG 结构 - 为每个节点指定条件概率表（CPT）

    Step 2：分解联合分布 - 利用图结构将联合分布分解为条件概率乘积 - \(p(x_1,\ldots,x_n) = \prod_i p(x_i \mid \text{pa}(x_i))\)

    Step 3：前向推理 - 给定某些变量的观测值，计算其他变量的边缘分布 - 方法：变量消元（Variable Elimination）或信念传播（BP）

    Step 4：逆向推断 - 给定观测结果，推断原因变量的后验概率（诊断推理） - 例如：已知交通拥堵，推断下雨的概率

    信念传播算法流程（树结构图）： 1. 选择任意节点作为根节点 2. 从叶节点向根节点传递消息（收集阶段） 3. 从根节点向叶节点传递消息（分发阶段） 4. 每个节点根据收到的所有消息计算边缘分布

    参数学习方法： - 完全数据：最大似然估计（MLE），直接统计频率 - 隐变量：EM 算法（如 HMM 的 Baum-Welch） - 结构学习：评分搜索（BIC/BDEScore）或条件独立性检验

    5. Python 最小实验¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib"])

    # === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

# 简单贝叶斯网络推断演示
# 场景：雨天(R) → 路湿(W) → 交通拥堵(T)
# 使用条件概率表进行前向推断
np.random.seed(42)

# 条件概率定义
P_rain = np.array([0.7, 0.3])  # P(R): 0=无雨, 1=有雨
# P(W|R): [P(W=0|R), P(W=1|R)]
P_wet_given_rain = np.array([[0.8, 0.2],   # R=0: 无雨时路湿概率0.2
                              [0.1, 0.9]])   # R=1: 有雨时路湿概率0.9
# P(T|W): [P(T=0|W), P(T=1|W)]
P_traffic_given_wet = np.array([[0.7, 0.3],  # W=0: 路干时拥堵概率0.3
                                 [0.2, 0.8]])  # W=1: 路湿时拥堵概率0.8

# 通过枚举计算联合分布 P(R, W, T)
joint = np.zeros((2, 2, 2))
for r in range(2):
    for w in range(2):
        for t in range(2):
            joint[r, w, t] = P_rain[r] * P_wet_given_rain[r, w] * P_traffic_given_wet[w, t]
print("联合分布 P(R, W, T) 各状态概率:")
labels = ['R=0,W=0,T=0', 'R=0,W=0,T=1', 'R=0,W=1,T=0', 'R=0,W=1,T=1',
          'R=1,W=0,T=0', 'R=1,W=0,T=1', 'R=1,W=1,T=0', 'R=1,W=1,T=1']
for lbl, val in zip(labels, joint.flatten()):
    print(f"  {lbl}: {val:.4f}")

# 条件推断：给定观测T=1（交通拥堵），推断P(R=1|T=1)
P_T1 = np.sum(joint[:, :, 1])  # 边缘化
P_R1_T1 = np.sum(joint[1, :, 1])  # P(R=1, T=1)
P_R1_given_T1 = P_R1_T1 / P_T1
print(f"\nP(有雨|交通拥堵) = {P_R1_given_T1:.4f}")
# 对比无条件概率
print(f"P(有雨)         = {P_rain[1]:.4f}")

# 可视化：不同观测下的后验概率对比
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
# 左图：P(R|T) 条件概率
P_T = np.sum(joint, axis=(0, 1))  # P(T)
P_R_given_T = np.zeros((2, 2))
for t in range(2):
    P_R_given_T[:, t] = np.sum(joint[:, :, t], axis=1) / P_T[t]
states = ['无雨', '有雨']
colors = ['#4ECDC4', '#FF6B6B']
x_pos = np.arange(2)
width = 0.35
axes[0].bar(x_pos - width/2, P_R_given_T[:, 0], width, color=colors[0], label='T=0(畅通)')
axes[0].bar(x_pos + width/2, P_R_given_T[:, 1], width, color=colors[1], label='T=1(拥堵)')
axes[0].set_xticks(x_pos)
axes[0].set_xticklabels(states)
axes[0].set_title('给定交通状况下的雨天概率', fontsize=13)
axes[0].set_ylabel('Probability')
axes[0].legend(fontsize=10)
axes[0].grid(True, alpha=0.3, axis='y')
# 右图：三种条件下的后验对比
P_W = np.sum(joint, axis=(0, 2))
P_W_given_T1 = np.sum(joint[:, :, 1], axis=0) / P_T[1]
P_W_given_R1 = P_wet_given_rain[1, :]
bar_data = [P_W, P_W_given_T1, P_W_given_R1]
bar_labels = ['无条件P(W)', 'P(W|交通拥堵)', 'P(W|有雨)']
for i, (data, lbl) in enumerate(zip(bar_data, bar_labels)):
    axes[1].bar(x_pos + i * 0.25 - 0.25, data, 0.25, label=lbl, alpha=0.85)
axes[1].set_xticks(x_pos)
axes[1].set_xticklabels(['路干', '路湿'])
axes[1].set_title('不同条件下的路湿概率', fontsize=13)
axes[1].set_ylabel('Probability')
axes[1].legend(fontsize=10)
axes[1].grid(True, alpha=0.3, axis='y')
plt.suptitle('贝叶斯网络前向推断与条件推断', fontsize=15)
plt.tight_layout()
plt.show()

    结果解释：代码构建了一个简单的三层贝叶斯网络（雨天→路湿→拥堵），展示了前向推断（从原因到结果）和逆向推断（从结果到原因）。可以看到，已知交通拥堵时，下雨的后验概率从先验的 0.30 上升到约 0.60——这正是贝叶斯网络"诊断推理"的力量。

    6. 常见误区¶

    误区 正确理解
    有向图的边表示因果关系 边表示条件依赖关系，不一定是因果。因果推断需要额外的假设（如 do-calculus）
    无向图和有向图可以随意互换 两者的独立性结构不同，某些条件独立性只能用有向图表达，某些只能用无向图表达
    信念传播在任何图上都能得到精确结果 BP 只在树结构图上精确；在有环图上需要使用 loopy BP，结果是近似的
    图模型的结构必须从数据中学到 结构可以由领域专家指定，也可以从数据学习；但结构学习是 NP-hard 问题
    HMM 只能处理离散状态 HMM 可以扩展到连续观测（如高斯 HMM）和连续状态（如卡尔曼滤波）
    配分函数 \(Z\) 总能计算 对于马尔可夫随机场，配分函数的计算通常是指数级的，这正是无向图推断困难的根源
    贝叶斯网络必须是无环的 贝叶斯网络的定义就是 DAG（无环）；但动态贝叶斯网络可以建模时序中的循环依赖

    7. 启发问题与创新拓展¶

    思考题¶

    1. 为什么卡尔曼滤波可以看作一种特殊的概率图模型？它的图结构是什么样的？（提示：链式有向图，状态-观测的两层结构）
    2. 在有环图上运行信念传播会怎样？为什么 loopy BP 在实践中往往效果还不错？
    3. 贝叶斯网络和神经网络有什么联系？深度学习中的"注意力机制"可以用图模型的语言描述吗？
    4. 如何用概率图模型表达"时间序列中的因果关系"？动态贝叶斯网络和 Granger 因果检验有什么区别？

    创新拓展方向¶

    • 概率编程：PyMC、Stan、Edward 等工具让用户用声明式语言定义图模型，自动推断
    • 因果推断：Judea Pearl 的结构因果模型（SCM），从观察数据推断因果关系
    • 图神经网络与概率图模型的融合：将消息传递的思想引入深度学习
    • 贝叶斯深度学习：将神经网络的权重视为图模型中的随机变量，进行不确定性建模
    • 因子图与信号处理：LDPC 码、Turbo 码的编译码都可以用因子图统一描述
    • 时序图模型：隐马尔可夫模型→状态空间模型→深度状态空间模型的演进
