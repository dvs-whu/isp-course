    阵列信号模型¶

    一、学习定位¶

    📚 拓展知识点说明

    定位：本知识点是本科课程的拓展内容，适合学有余力的同学深入学习。 一句话理解：多个麦克风/天线排成阵列，利用空间采样获取信号的方向信息。 前置要求：信号与系统基础、线性代数 与本科课程的关系：信号与系统基础在空间维度的扩展

    二、通俗入口¶

    直觉类比¶

    想象你站在广场上，有人在远处喊你的名字。你有两只耳朵，声音到达左耳和右耳的时间/相位略有不同——大脑正是利用这个"相位差"判断声音来自哪个方向。阵列信号模型就是把"两只耳朵"推广到 \(M\) 个天线，用数学精确描述这种空间相位差。

    均匀线阵（ULA）几何示意图¶

    信号从 \(\theta\) 方向以平面波到达均匀线阵，各阵元因路径差产生相位延迟 \(e^{-j2\pi md\sin\theta/\lambda}\)。

    ![[assets/array_signal_model_ai.png]]

    图：阵列信号模型：均匀线阵的阵列流形、方向矩阵与空间采样几何关系。

    三、严谨核心¶

    定义与概念¶

    阵列信号模型 描述空间中多个传感器（天线）接收来自不同方向信号源的数学模型。阵列信号处理的核心假设是：信号到达不同传感器时存在由传播路径差引起的相位差，这一相位差与来波方向相关。

    基本假设： - 远场窄带信号模型 - 信号源数为 \(D\)，阵元数为 \(M\) - 信号通过不同路径到达各阵元，产生不同的时延/相位

    阵列输出向量：

    \[\mathbf{x}(t) = \sum_{i=1}^{D} \mathbf{a}(\theta_i) s_i(t) + \mathbf{n}(t) = A \mathbf{s}(t) + \mathbf{n}(t)\]

    其中 \(\mathbf{a}(\theta_i)\) 为方向 \(\theta_i\) 对应的导向矢量（steering vector），\(A = [\mathbf{a}(\theta_1), \dots, \mathbf{a}(\theta_D)]\) 为阵列流形矩阵。

    对于均匀线阵（ULA），阵元间距为 \(d\)：

    \[\mathbf{a}(\theta) = [1, e^{-j2\pi d\sin\theta/\lambda}, \dots, e^{-j2\pi (M-1)d\sin\theta/\lambda}]^T\]

    核心公式¶

    \[\mathbf{x}(t) = A \mathbf{s}(t) + \mathbf{n}(t)\]
    \[\mathbf{a}(\theta) = \left[1, \, e^{-j2\pi \frac{d}{\lambda}\sin\theta}, \, \dots, \, e^{-j2\pi \frac{(M-1)d}{\lambda}\sin\theta}\right]^T\]

    协方差矩阵：

    \[R_{xx} = E[\mathbf{x}(t)\mathbf{x}^H(t)] = A R_{ss} A^H + \sigma_n^2 I\]

    阵列类型对比¶

    阵列类型 几何结构 导向矢量形式 自由度 优缺点
    均匀线阵（ULA） 等间距直线 Vandermonde 向量 \(M-1\) 结构简单，角度估计有模糊
    均匀圆阵（UCA） 等间距圆周 Bessel 函数展开 \(M-1\) 360°全向覆盖，无角度模糊
    L型阵 两正交线阵 分块结构 \(\geq 2M-2\) 可估计二维角度
    平面阵（URA） 矩形网格 Kronecker 积形式 \((M_x-1)(M_y-1)\) 可估计方位角+俯仰角
    稀疏阵 非均匀间距 与结构相关 可超过 \(M-1\) 扩展孔径，但有模糊问题

    典型应用¶

    • 雷达目标检测与参数估计
    • 无线通信中的空分多址（SDMA）
    • 声纳目标定位
    • 射电天文中的信号接收

    四、方法流程¶

    从物理到数学的建模流程¶

    1. 确定阵列几何：选择阵元数 \(M\)、阵元间距 \(d\)、排列方式（线阵/圆阵/平面阵）
    2. 建立坐标系：定义来波方向 \(\theta\) 的度量基准（阵列法线方向）
    3. 推导导向矢量：根据几何关系计算各阵元的相位延迟，得到 \(\mathbf{a}(\theta)\)
    4. 组装阵列流形矩阵：对所有感兴趣方向 \(\theta_1, \dots, \theta_D\)，拼接得到 \(A\)
    5. 写出阵列输出模型：\(\mathbf{x}(t) = A\mathbf{s}(t) + \mathbf{n}(t)\)
    6. 估计协方差矩阵：用 \(L\) 个快拍的样本估计 \(\hat{R} = \frac{1}{L}\sum_{l=1}^{L}\mathbf{x}(l)\mathbf{x}^H(l)\)

    关键设计选择¶

    • 阵元间距：\(d = \lambda/2\) 是最常用选择，避免空间混叠（栅瓣）
    • 阵元数 \(M\)：越大空间分辨率越高，但成本和计算量增加
    • 快拍数 \(L\)：协方差矩阵估计精度随 \(L\) 增大而提高

    五、Python 最小实验¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib"])

    import numpy as np
import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False

# === 实验：ULA阵列流形可视化 ===
M = 16  # 阵元数
d_over_lambda = 0.5  # 阵元间距 = 半波长
theta_range = np.linspace(-np.pi/2, np.pi/2, 361)

# 导向矢量函数
def steering_vector(theta, M, d_lam):
    m = np.arange(M)
    return np.exp(-1j * 2 * np.pi * d_lam * m * np.sin(theta))

# 方向矩阵（阵列流形）
A = np.array([steering_vector(th, M, d_over_lambda) for th in theta_range]).T

# 可视化
fig, axes = plt.subplots(1, 3, figsize=(15, 4))
theta_deg = np.degrees(theta_range)

# 左图：不同方向的阵元响应幅度
resp = np.abs(A[:, ::36])  # 每隔5度取一个方向
for i in range(resp.shape[1]):
    axes[0].plot(range(M), resp[:, i], label=f'θ={theta_deg[i*36]:.0f}°')
axes[0].set_title('各阵元对不同方向的响应幅度', fontsize=13)
axes[0].set_xlabel('阵元索引')
axes[0].set_ylabel('Response Amplitude')
axes[0].legend(fontsize=9)
axes[0].grid(True, alpha=0.3)

# 中图：导向矢量相位分布
theta_test = 30
a_30 = steering_vector(np.radians(theta_test), M, d_over_lambda)
axes[1].plot(range(M), np.angle(a_30), 'ro-', linewidth=2, markersize=6)
axes[1].set_title(f'θ={theta_test}°方向的导向矢量相位', fontsize=13)
axes[1].set_xlabel('阵元索引')
axes[1].set_ylabel('Phase (rad)')
axes[1].grid(True, alpha=0.3)

# 右图：阵列因子（波束指向0°）
w = steering_vector(0, M, d_over_lambda) / M
AF = np.abs(w.conj() @ A)
axes[2].plot(theta_deg, 20*np.log10(AF + 1e-10), 'b-', linewidth=2)
axes[2].set_title('阵列因子（波束指向0°）', fontsize=13)
axes[2].set_xlabel('角度 (度)')
axes[2].set_ylabel('Gain (dB)')
axes[2].set_ylim([-40, 0])
axes[2].grid(True, alpha=0.3)

plt.suptitle(f'ULA阵列流形 (M={M}, d=λ/2)', fontsize=15)
plt.tight_layout()
plt.show()

    实验要点：观察左图中不同方向信号在阵元上产生的不同响应模式；中图验证导向矢量的线性相位结构；右图看到均匀加权的阵列因子主瓣宽度约为 \(\lambda/(Md)\)。

    六、常见误区¶

    误区 1：阵元间距越大越好¶

    正解：间距超过 \(\lambda/2\) 会出现空间混叠（栅瓣），导致角度模糊。就像时域采样要满足奈奎斯特准则，空域采样同样有空间采样定理约束。

    误区 2：导向矢量只与方向有关¶

    正解：导向矢量同时依赖阵元间距 \(d\)、波长 \(\lambda\) 和阵列几何结构。相同方向在不同频率下的导向矢量不同，这就是宽带阵列处理的挑战来源。

    误区 3：协方差矩阵可以精确已知¶

    正解：实际中 \(R_{xx}\) 只能用有限快拍估计，估计误差直接影响波束形成和 DOA 估计性能。快拍数 \(L\) 至少需要 \(O(M)\) 量级才能保证估计质量。

    误区 4：远场假设总是成立¶

    正解：远场条件要求 \(r > 2D^2/\lambda\)（\(D\) 为阵列孔径）。对于大孔径阵列或近场场景（如室内麦克风阵列），需要使用近场球面波模型。

    七、启发问题与创新拓展¶

    思考题¶

    1. 如果阵元间距不是均匀的（稀疏阵），导向矢量的形式会如何变化？自由度会增加吗？
    2. 协方差矩阵 \(R_{xx}\) 的特征值分布能告诉我们信号源的数量吗？如何量化判断？
    3. 如果信号是宽带的而非窄带的，阵列信号模型需要做哪些修改？

    创新拓展方向¶

    • 深度学习 + 阵列模型：用神经网络学习从协方差矩阵到 DOA 的映射，绕过传统子空间分解（DeepMUSIC）
    • 稀疏阵列设计：互质阵、嵌套阵等非均匀阵列用少量阵元获得 \(O(M^2)\) 个自由度
    • 阵列校正：实际阵列存在互耦、位置误差、幅相不一致，鲁棒建模是工程关键
    • 动态场景建模：时变信道中导向矢量随时间漂移，需要自适应更新阵列模型

    跨领域联系¶

    • 信号与系统：阵列信号模型将一维时域信号扩展到空域，导向矢量本质上是空间域的复指数信号，与时域傅里叶基函数对应
    • 数字信号处理：协方差矩阵估计与特征分解是统计信号处理的基本工具，阵列模型为空域处理提供了统一的数学框架
    • 线性代数：阵列流形矩阵的结构（Vandermonde 矩阵）直接决定了后续波束形成与 DOA 估计的可行性
