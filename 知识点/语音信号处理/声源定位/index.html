    声源定位¶

    1. 学习定位¶

    本科导学定位

    周次：W10 语音AI处理应用 深度：了解即可（不纳入期末笔试） 关联实验和项目：项目二E3、H2和环境监测方向。重点理解TDOA的基本思想和GCC-PHAT方法。

    2. 通俗入口¶

    🎯 初学者简化说明

    一句话理解：声源定位就是用多个麦克风"听"声音到达的时间差，像蝙蝠一样算出声音从哪个方向来。

    你需要重点掌握的：

    1. TDOA（到达时间差）：两个麦克风收到同一声音的时间差 \(\tau\)，利用时间差和声速可以算出方向角
    2. 互相关/广义互相关：用信号互相关函数的峰值位置来估计 TDOA
    3. 麦克风阵列：多个麦克风协同工作，阵列的几何结构决定了定位精度和分辨率

    作为进阶了解的（期末不要求推导）：

    • MUSIC 算法的数学推导（特征子空间分解 + 空间谱搜索）
    • GCC-PHAT 加权的详细推导

    💡 一句话记忆法：TDOA="两麦克风的时间差定方向"，阵列="多个耳朵比一个听得准"

    典型应用： - 智能音箱的声源定位与跟踪（如Amazon Echo） - 机器人听觉系统 - 视频会议中的说话人跟踪 - 声学监测与安防系统

    3. 严谨核心¶

    数学对象与基本假设¶

    声源定位（Sound Source Localization）是利用麦克风阵列接收到的信号，估计声源在空间中的方向（DOA，Direction of Arrival）或三维位置的技术。

    核心概念： - 到达时间差（TDOA）：声源信号到达不同麦克风的时间差异 - 波束形成（Beamforming）：通过空间滤波增强特定方向的信号 - 空间混响：房间反射导致的多径效应影响定位精度 - 远场与近场：远场假设平面波，近场需考虑球面波

    核心公式¶

    TDOA与DOA的关系（两个麦克风，远场）：

    \[\tau = \frac{d \cos\theta}{c}\]

    其中 \(d\) 为麦克风间距，\(\theta\) 为声源方向角，\(c\) 为声速（约340 m/s）。

    广义互相关-相位变换（GCC-PHAT）：

    \[R_{12}^{PHAT}(\tau) = \int_{-\infty}^{\infty} \frac{X_1(\omega) X_2^*(\omega)}{|X_1(\omega) X_2^*(\omega)|} e^{j\omega\tau} d\omega\]

    TDOA估计：

    \[\hat{\tau} = \arg\max_\tau R_{12}^{PHAT}(\tau)\]

    延迟-求和波束形成输出：

    \[y(n) = \frac{1}{M} \sum_{m=1}^{M} x_m(n - \Delta_m)\]

    其中 \(\Delta_m\) 为第 \(m\) 个麦克风的补偿延迟。

    MUSIC算法的空间谱：

    \[P_{MUSIC}(\theta) = \frac{1}{\mathbf{a}^H(\theta) \mathbf{E}_n \mathbf{E}_n^H \mathbf{a}(\theta)}\]

    其中 \(\mathbf{a}(\theta)\) 为导向矢量，\(\mathbf{E}_n\) 为噪声子空间。

    适用条件和边界¶

    • GCC-PHAT在强混响和低信噪比环境下性能下降，但PHAT加权可部分缓解
    • 远场假设要求声源距离远大于阵列孔径（\(r \gg Md\)），近场模型需球面波
    • MUSIC要求源数 < 麦克风数，且需要足够多的快拍数估计协方差矩阵
    • TDOA方法的空间混叠：当麦克风间距 \(d > \lambda/2\) 时出现空间模糊

    4. 方法流程¶

    输入是什么： 多通道麦克风信号 \(\{x_1(n), x_2(n), \ldots, x_M(n)\}\)。

    处理步骤（GCC-PHAT方法）： 1. 对各通道信号做FFT，得到频域表示 \(X_m(\omega)\) 2. 计算互功率谱并做PHAT加权归一化 3. IFFT得到广义互相关函数 \(R_{12}^{PHAT}(\tau)\) 4. 峰值检测得到TDOA估计 \(\hat{\tau}\) 5. 由几何关系 \(\theta = \arccos(\hat{\tau} c / d)\) 反推方向角

    处理步骤（MUSIC方法）： 1. 采集多快拍数据，估计协方差矩阵 \(\hat{\mathbf{R}}\) 2. 特征值分解，分离信号子空间和噪声子空间 3. 在角度网格上搜索空间谱 \(P_{MUSIC}(\theta)\) 的峰值 4. 峰值位置对应各声源的DOA

    输出如何解释： 估计的声源方向角 \(\hat{\theta}\) 或三维位置，可配合极坐标图可视化。

    常见参数如何选择： - GCC-PHAT：FFT长度一般取信号帧长的2倍 - MUSIC：快拍数一般取 \(K \geq 2M\) 以保证协方差矩阵估计稳定 - 麦克风间距：\(d \leq \lambda_{\min}/2\) 避免空间混叠

    方法对比¶

    方法 原理 麦克风数 分辨力 抗混响能力 计算复杂度 适用场景
    GCC-PHAT 广义互相关峰值 \(\geq 2\) 中 较强 \(O(N\log N)\) 远场窄带/宽带
    延迟求和 BF 空间功率扫描 \(\geq 2\) 低 一般 \(O(MN)\) 简单场景
    MUSIC 子空间正交性 \(\geq 3\) 高 较弱 \(O(M^3)\) 远场多源
    SRP-PHAT 全局网格搜索 \(\geq 3\) 高 强 \(O(N_\theta N\log N)\) 混响环境
    基于深度学习 端到端映射 \(\geq 1\) 高 强 取决于网络 复杂声学环境

    5. Python最小实验¶

    运行前准备：本代码需要以下Python库，请确保已安装： - 本地运行：pip install numpy matplotlib - Pyodide环境：先运行下面的安装代码块

    # Pyodide环境：安装依赖库
import micropip
await micropip.install(["numpy", "matplotlib"])

    # === 环境预加载 ===
import numpy as np
import matplotlib
import matplotlib.pyplot as plt
matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Arial Unicode MS', 'DejaVu Sans']
matplotlib.rcParams['axes.unicode_minus'] = False

import numpy as np
import matplotlib.pyplot as plt

# 参数设置
c = 340          # 声速 (m/s)
d = 0.1          # 麦克风间距 (m)
fs = 16000       # 采样率
duration = 0.02  # 信号时长 20ms
theta_true = 60  # 真实声源方向角（度）

# 生成宽带信号（模拟语音脉冲）
t = np.arange(0, duration, 1/fs)
np.random.seed(42)
source = np.random.randn(len(t))  # 白噪声模拟语音

# 计算TDOA
tau_true = d * np.cos(np.deg2rad(theta_true)) / c  # 真实时间差
delay_samples = int(abs(tau_true) * fs)

# 生成两路麦克风信号（带时延）
mic1 = source.copy()
mic2 = np.zeros_like(source)
if tau_true >= 0:
    mic2[delay_samples:] = source[:-delay_samples] if delay_samples > 0 else source
else:
    mic1[delay_samples:] = source[:-delay_samples] if delay_samples > 0 else source

# 互相关计算TDOA
def gcc_phat(sig1, sig2, fs):
    """广义互相关-相位变换（GCC-PHAT）"""
    n = len(sig1)
    SIG1 = np.fft.rfft(sig1, n=2*n)
    SIG2 = np.fft.rfft(sig2, n=2*n)
    # PHAT加权
    cross_spec = SIG1 * np.conj(SIG2)
    cross_spec /= (np.abs(cross_spec) + 1e-10)
    cc = np.fft.irfft(cross_spec, n=2*n)
    # 移位到 [-N, N] 范围
    cc = np.concatenate([cc[n:], cc[:n]])
    lags = np.arange(-n, n) / fs
    return cc, lags

# 计算互相关
cc, lags = gcc_phat(mic1, mic2, fs)
# 估计TDOA（互相关峰值位置）
tau_hat = lags[np.argmax(cc)]
# 估计方向角
theta_hat = np.rad2deg(np.arccos(np.clip(tau_hat * c / d, -1, 1)))

# 绘图
fig, axes = plt.subplots(1, 3, figsize=(14, 4))
# 两路信号
axes[0].plot(t*1000, mic1, 'b-', alpha=0.7, label='麦克风1')
axes[0].plot(t*1000, mic2, 'r-', alpha=0.7, label='麦克风2')
axes[0].set_xlabel('Time (ms)'); axes[0].set_ylabel('Amplitude')
axes[0].set_title('双麦克风接收信号'); axes[0].legend()

# 互相关函数
axes[1].plot(lags*1e6, cc, 'k-')
axes[1].axvline(tau_true*1e6, color='r', linestyle='--', label=f'真实τ={tau_true*1e6:.1f}μs')
axes[1].axvline(tau_hat*1e6, color='g', linestyle=':', label=f'Estimated τ={tau_hat*1e6:.1f}μs')
axes[1].set_xlabel('Time Delay (μs)'); axes[1].set_ylabel('Cross-correlation')
axes[1].set_title('GCC-PHAT 互相关'); axes[1].legend()

# 极坐标显示方向估计
ax = fig.add_subplot(133, projection='polar')
ax.set_theta_zero_location('N')
ax.set_theta_direction(-1)
ax.plot(np.deg2rad(theta_true), 1, 'r*', markersize=15, label=f'真实方向 {theta_true}°')
ax.plot(np.deg2rad(theta_hat), 0.8, 'gs', markersize=10, label=f'Estimated DOA {theta_hat:.1f}°')
ax.set_title('声源方向估计', pad=20)
ax.legend(loc='lower right', fontsize=8)
plt.tight_layout()
plt.show()

    结果解释： - 左图：两路麦克风信号，可见时间延迟导致的波形偏移 - 中图：GCC-PHAT互相关函数，峰值位置对应TDOA估计值 - 右图：极坐标下真实方向与估计方向的对比，两者应高度吻合 - GCC-PHAT方法通过频域加权提升抗混响性能

    6. 常见误区¶

    公式误解： - TDOA公式 \(\tau = d\cos\theta/c\) 仅在远场平面波假设下成立——近场时需用球面波模型，TDOA与距离也有关 - GCC-PHAT中的PHAT加权不是"简单归一化"——它将互功率谱的幅度信息完全去除，只保留相位，从而在混响环境下锐化互相关峰值

    参数误解： - 麦克风间距不是越大越好——间距超过半波长时出现空间混叠（多个方向产生相同TDOA），对于4kHz语音信号，\(d \leq 4.25\) cm - MUSIC算法不能处理相干信号——当多径反射导致信号相干时，需要先做空间平滑预处理

    适用条件误解： - 双麦克风只能定位一维角度（锥面），不能确定三维方向——需要至少4个非共面麦克风才能做三维定位 - 深度学习声源定位不是"不需要麦克风阵列"——单麦克风方法利用频谱线索，但精度远不如多麦克风方法

    单位误解： - TDOA的单位是秒（或微秒），不是角度——需要通过几何关系才能转换为方向角 - 声速 \(c \approx 340\) m/s 是在20°C空气中的值，温度每升高1°C声速增加约0.6 m/s

    7. 启发问题与创新拓展¶

    一个反例或边界问题： 如果两个声源恰好对称地分布在阵列两侧（如 \(\theta\) 和 \(360°-\theta\)），双麦克风TDOA能否区分？答案是不能——\(\cos\theta = \cos(360°-\theta)\)，两个方向产生相同的TDOA。这说明：双麦克风存在前后模糊问题，需要额外的麦克风（至少3个非共线）或利用头部遮挡等物理线索来消除模糊。

    一个跨学科迁移场景： 水下声呐定位：声源定位的TDOA原理完全适用于水下声学——海底水听器阵列利用声波到达不同水听器的时间差来定位潜艇或鲸鱼。但水下环境更复杂：声速随深度变化（约1500 m/s，且有声道效应），需要将恒速模型改为分层声速模型。这是声源定位技术在海洋科学中的经典应用。

    一个可做成课程项目的小想法： 设计一个"GCC-PHAT vs MUSIC vs 深度学习"对比实验：在不同混响条件下（RT60从0到1秒），用房间冲激响应模拟器生成多通道信号，分别用三种方法估计DOA，对比定位误差。特别关注：(1) 混响对GCC-PHAT峰值锐度的影响；(2) MUSIC在源数估计错误时的行为；(3) 深度学习方法的泛化能力。
